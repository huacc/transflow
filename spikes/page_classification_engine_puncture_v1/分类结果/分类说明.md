# 分类结果总说明

本目录保存最近一次物化运行的页面分类结果。每个 `Pxxxx.pdf` 只出现一次，且与 `样本/` 中同名 PDF 的 SHA-256 完全一致；具体 run_id 见对应运行报告。各目录中的 `分类说明.md` 不参与 PDF 集合恒等校验。

## 分类树

```text
page.role
├─ cover
├─ contents
├─ end
├─ visual_only
└─ body
   └─ body.layout_owner
      ├─ table
      ├─ chart
      ├─ diagram
      ├─ anchored_blocks
      ├─ composite
      │  └─ body.composite.kind
      │     ├─ flow_text_table
      │     ├─ anchored_blocks_chart
      │     ├─ chart_table
      │     ├─ flow_text_chart
      │     └─ flow_text_diagram
      ├─ flow_text
      │  └─ body.flow.topology
      │     ├─ single
      │     ├─ multi
      │     └─ visual_anchored
      └─ freeform（归约器兜底，不是千问候选）
```

## 分类定义原则

类别表示页面承担的功能和主要翻译排版权，不按文件所属公司、报告名称、语种、内容语义或文字密度分类。一级类别互斥；正文二级按主要结构所有权选择，`flow_text` 再按工具拓扑选择单栏、多栏或视觉锚定。

## 统一筛选机制

1. 一级只判断页面功能：封面、目录、正文、结束页或纯视觉页。
2. 工程规则与千问多模态独立初判；两者一致时直接进入该类别。
3. `body.layout_owner` 若获得置信度不低于 `0.90` 的 PDF 原生直接表格证据，并且纯表格/复合页边界清晰，则直接采用规则并跳过千问初判。
4. 其余情况两者不一致或任一方不确定时，只针对当前节点调用一次细粒度千问复核。
5. 一级复核仍不确定时停止为 `INCONCLUSIVE/page.role`；本次运行没有此类页面。
6. 只有一级确定为 `body` 才进入二级，判断主要翻译排版权归属。
7. 二级为 `flow_text` 时只判断工具拓扑：`single/multi/visual_anchored`；段落、列表、编号和日期事件只作为工具参数。
8. 二级为 `composite` 时只判断共同拥有页面的工具组合：`flow_text_table/anchored_blocks_chart/chart_table/flow_text_chart/flow_text_diagram`。
9. 任一正文子节点复核仍不确定时进入 `body/freeform`，用于人工查看分类树缺口。

文件名、源路径和人工标签不发送给千问。语种、具体栏数、文字密度和内容模式不是独立目录节点。图片可用于视觉判断，但图片像素及图片内部内容不进入翻译、重排或修复。
