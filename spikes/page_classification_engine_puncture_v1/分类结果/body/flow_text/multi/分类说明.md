# body/flow_text/multi：多栏正文工具箱

## 定义

主体可编辑文字形成两条或更多并列栏道。每条栏道拥有稳定的 x 轴范围和阅读顺序；长段落、项目列表、日期事件或混合内容均使用同一栏道约束。

## 正例与反例

- 正例：三栏财务要点，一栏中可包含多个项目符号。
- 正例：三栏业务展望，一栏中可包含长段落和分组标题。
- 正例：上方横跨页面的照片，下方三栏日期事件。
- 反例：多个上下文字框共享相同左右边界，仍属于 `single`。
- 反例：表格单元格的列不属于正文栏，应在上一级判 `table`。
- 反例：无边框“术语—释义”逐行配对不是多栏正文，即使视觉上没有表格线也应判 `table`。

## 筛选机制

规则识别两个以上具有纵向覆盖的 x 轴文字簇，同时排除同行重复配对的无边框表格。千问检查栏间空白和阅读顺序，并忽略跨栏标题与图片区。工具箱必须锁定原栏，只允许栏内换行、扩容和推移，禁止跨栏串写。项目符号和段落分组只影响栏内 grouping，不改变工具箱。
