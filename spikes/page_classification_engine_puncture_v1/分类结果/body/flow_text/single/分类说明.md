# body/flow_text/single：单栏正文工具箱

## 定义

主体可编辑文字只有一条稳定栏道，从上到下推进。段落、项目符号、编号议程、日期事件或混合内容都可以进入本类；内容语义不改变工具箱。

## 正例与反例

- 正例：整页单栏主席报告、审计附注或管理层讨论。
- 正例：单栏股东大会通知，包含数字编号和字母子项。
- 反例：两条以上并列主体栏道属于 `multi`。
- 反例：文字强绑定于图片或有色视觉槽位，属于 `visual_anchored`。

## 筛选机制

规则聚类主体文字块的 x 轴位置并排除跨栏标题、页眉、页脚和脚注；千问确认只有一条主体栏道。工具箱可在该栏内执行文字测量、换行、纵向扩容和下游推移。`content_pattern`、`density_band` 和编号绑定作为参数，不另建目录。
