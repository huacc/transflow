# body/composite/flow_text_diagram：正文与结构图组合工具箱

## 定义

实质连续正文与主体结构图共同承载页面信息。结构图表达流程、组织、层级、关系或时间顺序，依靠节点和连接关系理解，不是数据坐标图表。

## 正例与反例

- 正例：上半页为多段制度说明，下半页为包含节点和箭头的流程图。
- 反例：正文旁是柱状图、折线图或饼图，应判 `flow_text_chart`。
- 反例：结构图仅是可忽略的小图标，应由 `flow_text` 主导。

## 筛选机制

千问检查实质正文和结构关系是否都不可忽略，并区分数据图表与流程/组织关系图。工具调度保护结构图像素、节点和连接关系，只处理结构图外的独立 PDF 文字对象。
