# body/composite/flow_text_table：正文与表格组合工具箱

## 定义

主体表格和表外正文都不可忽略。表格具有明确行列、表头和单元格数据；表外存在实质段落或分析说明，两部分分别需要表格工具与正文流工具处理。

表格可以有网格线，也可以是通过稳定列锚点和同行配对形成的无边框表格。

## 正例与反例

- 正例：上半页为跨多列财务表，下半页为两栏长段分析文字。
- 反例：页面几乎全部是财务表，表外只有标题和短脚注，应判 `body/table`。
- 反例：只有多栏正文而没有真实表格，应判 `body/flow_text`。

## 筛选机制

工程规则检查显式/无边框表格证据、表格面积和表外文字量；直接证据达到 `0.90` 且表外正文明显充足时直接判复合页，中间边界由千问确认两者是否都拥有独立排版权。工具调度先锁定表格区域，分别执行表格回填与表外正文排版，最后进行跨区域溢出检查。
