# end：分类边界与不变量

## 正例

- 文档结束功能明确，品牌落款、公司名、联系信息、认证标记或结束视觉占主导；
- 没有实质正文、财务表、图表或目录关系；
- 可包含原生联系文字，也可仅由图片、矢量轮廓和品牌视觉构成。

## 反例

- 最后一页仍有正文、附注、表格或图表，应交给对应 `body` 叶子；
- 只有无功能照片或空白，应为 `visual_only`；
- 报告标题、报告期和进入文档语义占主导时应为 `cover`。

分类结果是只读上游。本工具箱不以“最后一页”重新判类。

## 可翻译对象

- 原生 PDF 文字中的电话标签、地址、免责声明和其他结束页语义文字；
- 相邻联系行可合并为一个局部 `contact_block`，保持原信息顺序；
- 数字、链接、认证缩写和注册符号进入 `required_literals`。

## 不可翻译对象

- 图片、Logo、二维码和矢量轮廓内部文字；
- 公司名、品牌落款、纯 URL、邮箱和纯符号；
- 在中文报告中已经是英文的联系信息，或在英文报告中已经是中文的联系信息；
- 已同时给出源语言和目标语言的双语联系标识。

## 不可动约束

- 页框、旋转、背景图片、Logo、二维码和矢量装饰的 `locked_objects_sha256` 不变；
- 受保护原生文字的内容与 bbox 不变；
- 译文输出区域不得侵入受保护文字 bbox；
- 允许区域外渲染差异必须低于机械阈值；
- 无可翻译原生文字的页面必须字节级透传。

## 能力不足条件

- 有可翻译 region 但翻译 Provider 或字体不可用；
- Provider 丢失 region、返回空译文或删除 `required_literals`；
- 无法找到不触碰受保护对象的红删或文字安全区；
- 译文在允许字号下仍无法装入。
