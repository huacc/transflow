# cover：工具分类与调用流程

## 工具清单

| 工具 | 类别 | 输入 | 输出 | 失败语义 |
|---|---|---|---|---|
| `build_cover_template` | 模板构建 | `PageFacts`、源 PDF | `CoverTemplate` | 原生文字所有权不完整 |
| `build_cover_translation_request` | 请求构建 | 模板、源/目标语种 | `PageTranslationRequest` | 视觉页或无待译源文字 |
| `plan_cover_layout` | 布局 | 模板、外部译文 | `CoverLayoutPlan` | ID 不匹配或文字溢出 |
| `render_cover_candidate` | PDF 回填 | 源页、模板、布局 | 候选 PDF、机械证据 | 无安全红删区、字体或锁定对象失败 |
| `run_p9_page` | 页级编排 | 样本、Provider、字体 | 完整运行包 | 诚实路由到 process/product/capability 状态 |
| `run_p9_cover.py` | 批量入口 | manifest 与样本选择 | 批量摘要 | 任一页面未进入接受终态时返回非零 |

## 固定流程

```text
PageFacts
→ PageTemplateBuilder
  → 后绘图片 bbox 候选
  → 临时删字后的像素贡献确认
→ PageTranslationRequest
→ 外部 TranslationProvider / 已有译文工件
→ LayoutPlanner
  → 几何型双语伴随行去重
→ CoverPdfRenderer
→ CoverJudge
→ 完整复裁与证据落盘
```

## Codex 探索记录

探索遵循单页、三页、全量的递进顺序。失败运行保留：第 18 轮发现 span 重组误报，第 20 轮发现透明蒙版被误判为遮挡；修复均先补真实链路回归，再进入全量。

## 流程冻结条件

- development、regression、严格盲 holdout 均通过；
- 真实译文全量轮与已有译文重放轮通过，候选预览和布局计划一致；
- 所有排版页完成源/候选人工视觉签收；
- 前置 P8 Gate 为 PASS；
- 运行时反过拟合扫描通过并生成 `PromotionManifest`。

本轮产品证据满足前三项中的非盲部分，但 P8 和严格盲 holdout 仍阻塞，因此 `workflow_frozen=false`。
