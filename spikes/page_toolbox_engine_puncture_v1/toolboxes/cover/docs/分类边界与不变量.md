# cover：分类边界与不变量

## 正例

- 已由上游分类为 `cover` 的年度报告、财务报告或章节封面；
- 页面以背景图、Logo、装饰图形和少量稀疏标题为主；
- 原生文字可为单语或双语，可与图片文字并存。

## 反例

- 具有目录条目与页码导航关系的 `contents`；
- 以结束语、联系信息或品牌落款为主的 `end`；
- 正文、表格及正文复合页；
- 只在图片像素或矢量轮廓中存在文字且没有可见原生语义文字的页面。

## 可翻译对象

- 当前目标语种仍缺失的可见原生语义文字；
- 报告名、章节名、机构文字、期间说明、法律身份和股票代码标签；
- 翻译请求只声明容器 ID、源文字、阅读顺序和必须保留的数字字面量。译文语义质量由外部翻译服务负责。

## 不可翻译对象

- 图片内文字、Logo、矢量轮廓文字和纯装饰；
- 纯数字、日期、纯 URL、邮箱及其他非语义字面量；
- 页面已有的目标语言伴随行；
- 被后续不透明内容完全覆盖、对最终渲染没有任何像素贡献的隐藏文字层。

## 不可动约束

- 背景、图片、Logo、颜色块、矢量装饰及图片内文字保持；
- 受保护文字的非空白字形序列和逐字形坐标保持；
- 稀疏标题的左、右或中心锚点、层级和阅读顺序保持；
- 既有目标语言伴随行不重绘，重复源行只按相对几何关系移除；
- 纯视觉页不调用翻译服务，并保持源 PDF 字节一致。

## 能力不足条件

- 原生文字所有权不完整；
- 译文容器 ID、顺序或数字字面量不满足请求合同；
- 找不到不触碰受保护文字的安全红删区域；
- 译文在允许区域和最小字号内仍无法装入；
- 必需字体不存在或未嵌入；
- 锁定对象、允许区域外像素或视觉锚点发生硬变化。

## 反过拟合边界

运行时代码只能使用当前 PDF 的对象事实、可见像素贡献、脚本方向、字号和相对几何；不得出现样本 ID、公司名词表、固定标题、已知年份或样本专属坐标。样本事实只允许出现在测试、清单和报告中。
