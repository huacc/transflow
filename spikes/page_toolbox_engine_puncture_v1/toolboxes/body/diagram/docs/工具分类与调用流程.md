# body.diagram 工具分类与调用流程

1. `shared_pdf_kernel.extract_page_facts` 提取原生文字、图片、矢量和不可动物哈希。
2. `diagram_template_builder` 按当前页几何识别节点候选、连线段和局部文字 owner。
3. 无可翻译原生文字时进入字节透传；否则生成一次页级翻译请求。
4. 翻译 Provider 只返回 `container_id → translated_text`，不参与排版。
5. `diagram_layout_planner` 先区分正文流、原节点框、地图卡片/锚点和连线走廊，再在各自有限档位内试排；不移动或扩张节点边界。
6. `diagram_pdf_renderer` 只删除已登记源文字；与保护符号发生小幅纵向边缘重叠时裁剪擦除区，随后回填译文并复核锁定对象、拓扑签名、保护文字和 owner 边界。
7. `diagram_quality_judge` 分开输出 Process、Product 和 Capability 结论，并保存源页、候选页、并排图及机器证据。无论 PASS/FAIL 都物化 `candidate.pdf`；失败 owner 保留源文。
8. 批处理可用 `--workers` 做页面级并发，主线程每页完成后原子更新 `reports/progress.json`；监控以进度文件和 case 产物为准。

拓扑恢复只依赖 PDF 几何：节点来自包含原生文字的最小闭合/填充图形；连接关系来自线段端点与节点边界的邻近关系；箭头方向仅在端点附近存在直接填充箭头事实时声明，否则记录为 `undirected`。
