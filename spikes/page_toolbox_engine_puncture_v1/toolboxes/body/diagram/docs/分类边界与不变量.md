# body.diagram 分类边界与不变量

## 正例

- 组织架构、治理层级、流程、树状关系、时间线和其他节点—连线示意图；
- 节点或标签文字是独立 PDF 原生文字，可以从 PDF 事实定位；
- 节点边界、连接线、箭头和空间顺序可以从矢量或图像对象事实锁定。

## 反例

- 数据图表属于 `body.chart`；
- 真实行列单元格属于 `body.table`；
- 正文与示意图共同主导且所有权不可拆分时属于 `body.composite.flow_text_diagram`；
- 图片化示意图内部文字不是可编辑节点文字，不进入翻译请求。

## 不变量

- 节点边界、层级、连线端点、箭头方向、图片、颜色和空间关系不可改变；
- 每个原生文字对象必须且只能属于一个翻译容器或保护集合；
- 节点译文只能写入原节点的完整安全内框；地图卡片、图例和局部锚点必须保持原图语义映射；
- 页面标题、正文说明、页眉页脚可以使用当前页事实证明安全的正文宽度和纵向留白，不受源字形窄框限制，但不得越过正文边界、其他 owner 或连线走廊；
- 数字、日期、代码、比例和其他结构标识由当前页正则与几何证据保护，不使用实体词典；
- 最低可读字号仍无法容纳时返回 `CAPABILITY_FAILED`，不扩大节点或重布线；无论终态如何都产出 `candidate.pdf`，失败 owner 保留源文并写入门禁清单；
- 没有可翻译原生文字时字节透传。
