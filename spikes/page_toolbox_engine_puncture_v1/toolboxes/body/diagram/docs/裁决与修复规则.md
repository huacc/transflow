# body.diagram 裁决与修复规则

## 硬失败

- 图片或矢量哈希变化、拓扑签名变化：`PROCESS_FAILED`；
- 保护文字丢失或移位、owner 外出现写入：`PROCESS_FAILED`；
- 节点译文越界、覆盖新增连线、原文残留或标签错属：`QUALITY_FAILED`；
- 字体缺字、节点在最低可读字号仍放不下、安全删除区域不存在：`CAPABILITY_FAILED`。

## 有限修复

每个容器只允许有限档位试排：

- 正文段落优先源字号及 1.55/1.45/1.35 舒展行距；接近连线时先守源纵向带，再有限降到 0.92/0.84/0.76/0.68；
- 普通标题、页眉页脚和独立说明可使用正文安全宽度，同一水平带的并列标签按源锚点对称分配走廊；
- 节点先守源锚点，确实放不下时可使用原节点完整安全内框，并有限降到 0.60/0.52；节点边界不扩张、不移动；
- 地图卡片使用原图像框，地图内普通标签保持坐标锁定；极小单字只允许围绕源中心做局部锚点扩展；
- 同 owner 的重复叠印文字先合并为一个翻译单元；保护符号只在可保留至少 80% 源擦除区时允许纵向安全裁边。

接受修复必须同时满足：目标容器可容纳、节点/连线/箭头签名不变、保护文字不变、其他 owner 不退化。否则返回能力不足或产品失败，不继续开放式搜索。

失败不会取消 PDF 物化。布局或质量门禁失败时，已安全回填的 owner 保留译文，失败 owner 保留源文，`candidate.pdf` 标记为部分审阅候选，同时在 `gate_failures.json` 列出失败码、容器和修复处置。
