# body.anchored_blocks：分类边界与不变量

## 正例

- KPI、治理、奖项、业务卡片、指标块和图文说明块彼此独立，文字围绕固定视觉锚点组织；
- 每个块有自己的边界、背景或图像关系，不应按连续多栏正文串联；
- 页面页眉、页脚中的原生语义文字属于其自身锚定块。

## 反例

- 连续段落跨栏流动属于 `body.flow_text.multi`；
- 由行列网格定义阅读关系的内容属于 `body.table`；
- 目录关系属于 `contents`，图表语义属于 chart/composite leaf；
- 只有图片或扫描文字而无可编辑原生文字时，不在本工具箱内做 OCR 或图像重绘。

分类结果来自只读上游，本工具箱不重新分类。

## 可翻译对象

- 块内原生标题、说明、段落、标签以及语义页眉/页脚；
- 同一语义段内可安全合并、且样式与阅读顺序一致的原生文字片段；
- 每个对象必须通过 `block_owner_id/container_id` 回填到原块。

## 不可翻译对象

- 图片、Logo、证书和矢量图形内部文字；
- 纯页码、纯数字、百分比、日期、货币值、代码和其他标识；
- 微型装饰文字、纯符号、URL 等不具备可安全翻译语义的对象。

## 不可动约束

- 每个原生文字对象恰好属于一个翻译容器或保护集合；
- 块边界、左上锚点、块顺序、图像、背景、边框和层叠关系不变；
- 译文只能进入本容器 `allowed_bbox`，不得借用相邻块或表格/卡片单元空间；
- 原文居中时译文仍居中；同语义样式保持相同字号，标题与正文的字号层级比例保持；
- 保护对象内容与 bbox 指纹不变，允许区域外像素变化率必须为零；
- 修复仅重算目标块，完整复裁仍检查所有块。

## 能力不足条件

- 页面没有可定位原生文字，或没有可翻译语义文字；
- Provider、字体或必保字面量合同不可用；
- 译文在允许的换行、行距和字号范围内仍无法装入本块；
- 无法在不触碰保护对象、相邻块或视觉锚点的前提下完成红删与回填。
