# body.anchored_blocks：工具分类与调用流程

## 工具清单

| 工具 | 类别 | 输入 | 输出 | 失败语义 |
|---|---|---|---|---|
| `build_anchored_blocks_template` | 叶子模板 | `PageFacts`、源 PDF | `BlockOwner`、翻译容器、保护集合 | 原生/可翻译文字缺失或 ownership 不完整时 capability fail |
| `build_anchored_translation_request` | 翻译合同 | 模板、语言方向 | 带 owner/container ID 与必保字面量的单页请求 | ID 或翻译单元不完整时 capability fail |
| `RequiredLiteralRetryProvider` | 翻译护栏 | 请求、主/重试 Provider | 经字面量、残留、占位符审计的译文 | 重试后仍不完整时 capability fail |
| `plan_anchored_layout` | 叶子布局 | 模板、译文、字体 | 块内 placement 与 Finding | 溢出、碰撞、字号层级漂移时 quality fail |
| `render_anchored_candidate` | 机械渲染 | 源页、模板、placement | 候选 PDF、渲染证据 | 保护对象或允许区域外内容变化时 process fail |
| `run_p11_page` | 固定编排 | 单页和 Provider | 自包含运行包、过程/产品结论 | 显式 `PAGE_PASSED`、`QUALITY_FAILED`、`CAPABILITY_FAILED` 或 `PROCESS_FAILED` |

## 固定流程

```text
source snapshot → PageFacts → AnchoredBlocksTemplateBuilder
→ PageTranslationRequest → guarded TranslationProvider
→ AnchoredLayoutPlanner → PDF renderer
→ ownership/protected/literal/residue/font/collision/outside-region judge
→ PAGE_PASSED | QUALITY_FAILED | CAPABILITY_FAILED | PROCESS_FAILED
```

布局适配只在当前容器的 `allowed_bbox` 内尝试有限换行、行距和字号，并可局部消解同 owner 碰撞；不得移动相邻块或建立全页万能 planner。

## 探索与冻结记录

1. 先以 development 页打通 owner、容器、保护项和真实千问；
2. 扩展非 holdout 样本，冻结结构画像、必保字面量、布局 profile 和裁决；
3. `runs/54_non_holdout_final_qwen` 得到 23 个 `PAGE_PASSED`，另 1 页诚实路由为预期 capability failure；
4. 原始盲运行 `runs/57_all_blind_qwen` 的 holdout 为 4/6，失败后仅做结构驱动的通用修复；
5. 页眉/页脚最终单页回归为 `runs/64_header_footer_single_recorded`，并重新冻结实现指纹。

## 流程冻结条件

工具函数、提示词、样本清单和测试文件全部进入冻结指纹；固定译文/recorded 与真实千问使用同一 runtime。由于修复后的 holdout 已非盲，当前可用于回归但不足以正式晋级。
