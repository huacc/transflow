# 工具分类与调用流程

## 确定性流程

1. `shared_pdf_kernel.extract_page_facts` 读取原生文字、图片、矢量与页面几何；
2. P4 模板和 P13 模板只作为基础事实解释器，P17 在 composite 边界内重新建立唯一所有权，并在移除 chart owner 后重新连接被图表文字打断的正文续接；
3. 长宽、对齐、文字密度、字号/颜色突变和图表关联共同确定 FlowBand、chart guard 与共享集合；普通字重的误标题可按通用续接条件恢复，粗体标题仍保持独立；
4. P17 按最终阅读顺序生成一个 `PageTranslationRequest`；图片文字和受保护对象不进入请求；
5. 每个已观察 FlowBand 选择 P4 单流有限 profile；图表文字选择 P13 有限 slot/profile；
6. 合并两个子规划的 placement，以选中 placement 的实际字形 bbox 复裁 FlowBand、图表 guard、跨 owner 碰撞与行首标点；
7. 从源 PDF 一次删除已拥有的原生文字并一次写入全部译文；
8. 比较锁定视觉签名、受保护文字、实际字形、嵌入字体、页外变化和候选可读性；
9. 输出 `PAGE_PASSED`、`CAPABILITY_FAILED`、`QUALITY_FAILED` 或 `PROCESS_FAILED`。

## 字体选择

目标为简体中文时使用微软雅黑及其粗体；目标为英文时使用 DejaVu Sans 及其粗体。选择依据是目标语言字符覆盖和 PDF 重新抽取的字符映射稳定性，不依据样本 ID。每个实际使用的字体资源必须嵌入候选 PDF，缺字直接失败。

## 翻译提供方

千问只返回与请求 ID 一一对应的译文，不返回坐标、字号或排版指令。原始 bundle、原始机械校验和提供方定点重试 audit 分别保存。长单元在两次整单元定点重试仍被截断时，可以按句读分段翻译；分段请求每批最多 3 项，并必须遮蔽并恢复字面量标记、合回原 owner/container ID，再执行整单元校验。坏 JSON、连接读中断、超时和既有瞬态 HTTP 状态只做有限重试。短单元不得用分段绕过失败。只有 audit 明确确认的专名 ID 可以从“目标中文中的英文残留”中豁免；其他残留、缺字面量、数量级错误、占位输出、不完整译文或不同源文共享同一长译文仍失败。

## 诊断候选

完整译文已经到达但安全布局失败时，工具箱可以把问题容器放入页内诊断槽，生成可读的翻译审阅 PDF。诊断槽不改变产品裁决，报告必须同时写明原始失败、`diagnostic_candidate=true` 和 `product_acceptance=false`。
