# 裁决与修复规则

## 有限修复顺序

正文只按冻结的 P4 profile 顺序尝试：自然字号/行距、压缩段距/行距、再逐级缩放到 72% 下限。每次尝试都重新检查 FlowBand、图表 guard 和行首闭合标点；首个完整通过的 profile 才能进入产品渲染。

图表文字只使用 P13 的源槽、相邻视觉对象形成的局部安全走廊、安全换行和有限字号 profile。同组成员逐一量测，共同 profile 不得因一个不可容纳成员而丢弃其他可容纳成员；重复年份等独立标签不得合并。不得移动图表构件、改变标签 owner、把多项标签合并成正文或删除语义换取通过。

跨 owner 碰撞使用选中 placement 渲染后的实际字形 bbox，不使用最大允许槽位推断碰撞。页脚、脚注或下标只有与正文栏道横向相交时才构成 FlowBand 下边界；同一父级及满足通用几何条件的相邻普通字重续接不拆成另一条正文流，有限左缘抖动不得制造两个语义 owner，粗体标题不参与该合并。

## 终态

- `PAGE_PASSED`：翻译、布局、渲染、不可变检查和产品裁决全部通过；
- `CAPABILITY_FAILED`：翻译合同失败、字体缺字、P4/P13 最小可读 profile 仍溢出，或正文无法留在 FlowBand；
- `QUALITY_FAILED`：流程可执行但出现跨 owner 碰撞、字形越槽、图表构件碰撞或其他产品质量 Finding；
- `PROCESS_FAILED`：源页/候选不可读、结构签名错配、渲染器异常或不可动对象实际变化。

`CHART_DATA_VISUAL_CHANGED`、`CHART_PROTECTED_TEXT_CHANGED` 和 `CHART_OUTSIDE_TEXT_REGION_CHANGED` 属于流程失败，不能降级成普通质量问题。

## 禁止修复

- 不按文件名、样本 ID、发行人、页码、正文主题或绝对坐标添加分支；
- 不翻译图片内文字，不 OCR，不重画图表；
- 不移开图表给正文腾位置，不让图表标签进入正文流；
- 不删除译文、不改数字/单位、不缩到冻结下限以下；
- 不把一个容器的长译文复制给另一段不同源文，也不让这种重复结果进入布局；
- 不把诊断 PDF、源页复制或“脚本运行成功”写成产品通过；
- 不因当前样本未触发多栏正文就宣称 P5 能力或形式晋级完成。
