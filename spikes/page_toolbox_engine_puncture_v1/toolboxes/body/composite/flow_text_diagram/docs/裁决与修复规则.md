# 裁决与修复规则

## 裁决顺序

1. 翻译完整性：ID 集合、必保字面量、目标语言和长文本完整度。
2. 子工具能力：正文计划和示意图节点文字是否能适配。
3. owner 边界：正文/共享文字是否新进入示意图结构区，是否产生跨 owner 或 flow/shared 之间的新增碰撞。
4. PDF 机械合同：图形拓扑、受保护文字、字体嵌入和允许区域外变化。
5. 产品裁决：没有 process/capability 问题后，才评估剩余 hard quality finding。

## 终态

- `PAGE_PASSED`：过程与产品均通过。
- `CAPABILITY_FAILED`：缺少正文/节点适配、字体、翻译完整性或安全擦除能力。
- `QUALITY_FAILED`：候选可生成，但出现跨 owner 碰撞、阅读关系破坏等产品问题。
- `PROCESS_FAILED`：源页、锁定图形、受保护对象或证据合同失真。

## 修复接受

当前仅允许基础叶已经注册的确定性排版档位和 P18 的诊断物化。源字形 bbox 只提供锚点；节点内可用空间由节点安全内框、标题/相邻 owner 阻挡物及最终字形包络共同决定。向安全空白扩展后仍须用最终字形框复核相邻平行槽位和受保护结构，不得以单框 fit 换取组内碰撞。组级方向继承源页后，只能在该方向内搜索列数、自然换行、字号和行距。源 span 中断且中间保留结构符时，两侧标题必须使用独立安全区。原生图片参与 flow/shared 的 blocker 与 hard collision 裁决；页面普通字体中重复出现的可信大写缩写不得被 CID 恢复改写。最终实排还要拒绝 CJK 闭合标点行首、开合标点行尾和拉丁词内断行。任何正式修复必须同时满足：目标 finding 改善、没有新增 hard finding、锁定对象与拓扑不变、全页重新裁决。诊断槽只用于让返回译文可见，始终携带 hard finding，不能成为产品 PASS。
