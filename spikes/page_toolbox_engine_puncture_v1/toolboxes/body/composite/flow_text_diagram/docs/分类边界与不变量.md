# 分类边界与不变量

## 正类

页面同时包含实质正文阅读流与承担主要信息的示意图。示意图可以是组织结构、流程、职责层级、治理结构、关系框或由连接线/箭头表达的结构，即使没有封闭节点也可以成立。

## 非本叶

- 只有正文装饰图标：路由到 `body.flow_text.single` 或 `body.flow_text.multi`。
- 示意图是唯一主体，正文仅是标题或极短说明：路由到 `body.diagram`。
- 主体是数据图表、表格或图片文字：分别路由到对应基础叶或 composite 叶。

## 硬不变量

1. 每个源文字对象恰好属于 `flow`、`diagram`、`shared` 或 `protected` 之一。
2. 一页只发一次翻译请求，返回项只通过稳定 composite container ID 回填。
3. 正文不得进入节点或跨过示意图结构区；节点文字不得进入正文阅读流。
4. 节点、连线、箭头、层级、方向、图片和绘图对象保持不变。
5. 正文与示意图计划必须从源页一次合成，禁止两个子工具箱顺序改写同一 PDF。
6. capability、quality、process 三类失败分别记录；诊断候选不转写为产品 PASS。
7. 同一框架组的文字排布方向继承源页，目标语言适配不得逐框改向；单列放不下时只允许在原方向内换行或分列。
8. 被保护的分隔符、标题栏和节点边界切分局部安全区；译文不得跨越这些结构把两侧语义重新合成一框。
9. 原生图片是 flow/shared 重排的硬阻挡物；新增字形不得覆盖证书、照片、徽标等非背景位图。

## 反过拟合边界

运行时代码不得包含样本 ID、文件名、公司名、固定句子或绝对页面坐标分支。结构区检测只使用相对页面尺度、节点/连接线直接事实、几何聚类和 owner 关系；测试需要覆盖页面缩放及中英文方向变化。
