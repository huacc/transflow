# 工具分类与调用流程

## 调用链

1. `shared_pdf_kernel.extract_page_facts` 提取不可变页面事实。
2. P14 diagram template builder 提取节点、连接线和候选标签。
3. P18 template builder 从节点或高密度结构线聚类确定主体示意图区域，并先认领示意图原生文字。
4. 剩余文字事实交给正文侧；能稳定形成两/三栏时使用 P5 multi 模板，否则使用 P4 single 模板。
5. P18 对正文容器再区分 `flow` 与 `shared`，其余对象进入 `protected`，然后验证所有权穷尽且无重复。
6. 全部可翻译容器按页面阅读顺序组成一个 `PageTranslationRequest`。
7. 翻译结果按 composite ID 切回 flow 与 diagram 子计划。
8. 两类 placement 合并为一份 P18 render plan，并从源页一次完成文字擦除和回填。
9. 同时复裁正文边界、跨 owner 碰撞、节点/连接线拓扑、受保护文字、字体嵌入和区域外变化。

## 共享与私有

- 共享：事实提取、字体探测、PDF 渲染、哈希和机械证据。
- P4/P5 子能力：正文分块、单流/多流判断和正文排版计划。
- P14 子能力：节点、连线、图形拓扑和节点内文字适配。
- P18 私有：四类所有权、结构区聚类、共享标题/图注归属、全页一次翻译、跨区域裁决和 composite Gate。

P18 不把自己的 planner、阈值或样本规则写回任何基础叶。
