# 裁决与修复规则

## 终态

- `PAGE_PASSED`：请求 ID 完整、所有权唯一、译文已物化、布局和锁定对象校验通过、候选 PDF 存在。
- `CAPABILITY_FAILED`：缺少合法 chart/table owner，或当前安全容纳合同无法处理；必须记录 failure owner 和错误码。
- `QUALITY_FAILED`：owner 与能力存在，但候选产品违反溢出、碰撞、字形、表格线或锁定视觉约束。
- `PROCESS_FAILED`：候选违反不可变对象合同，或事实提取、布局、渲染、质量裁决发生非产品能力异常；必须保留失败阶段 owner。

批次缺少 `PAGE_TOOLBOX_QWEN_API_KEY` 属于 provider 能力失败，产品结论为 `NOT_REACHED`，不能换成占位译文继续跑。

只有在完整译文已经建立后才允许发布 `output/candidate.pdf`：正常页输出正式候选；已有完整译文但布局、能力或质量失败时，尽量物化全部译文并输出翻译诊断候选。模板、provider 或提前流程失败且没有完整译文时不得复制源页冒充候选，`candidate_artifact.json` 必须记录 `artifact_kind=NO_TRANSLATED_CANDIDATE`。诊断件统一记录 `diagnostic_candidate=true`、`product_acceptance=false`、真实终态和失败码。PDF 存在只证明问题可见，绝不改变失败裁决。

## 固定裁决顺序

1. P13 建立图表模板和不可动物；
2. P6 建立表格对象、行列、单元格和数值证据；
3. P6 明确拥有的对象从 P13 邻近 owner 中移出；
4. 建立 table region，并识别紧邻标题、单位等 shared owner；
5. 对全部原生文字做 chart/table/shared/protected 精确一次分区；
6. 构造一个页级翻译请求并校验无重复、无遗漏；
7. 分 owner 规划，但统一检查 table region、chart region 和 placement 碰撞；
8. 单次渲染后复核锁定视觉、表格线、字形、越界和相交。

## 当前一般化修复

- 连续至少三行出现相同语义间隔时，恢复无竖线表格的语义子列；
- 尾部行只有在至少占用两列且包含至少两个数值对象时才扩入表格；
- 同列上下堆叠的表头碎片先合并，再进入翻译；
- 表格上方邻近标题或单位显式归为 shared；
- P13 没有 table seed 时，选择同时含语义与数值单元的最大连续行带；
- fallback 最多吸收两个前置表头行，绝不吞入尾部图表行。

这些规则只依赖页面结构，不依赖样本 ID、证券号、页码、已知文字或某个绝对坐标。

## 不允许的修复

- OCR 图片文字并覆盖回 PDF；
- 为某个样本 ID、文件名、页码或证券号写分支；
- 把未识别的表格当 flow text；
- 为通过锁定校验而移动表格线、图表数据视觉或受保护数值；
- 分别渲染叶工具后再叠加两个候选 PDF；
- holdout 暴露后修改冻结规则，却继续沿用原 freeze revision。

任何跟踪文件变更都会使当前 freeze revision 失效，必须建立新 revision，并重新执行初始 1 页和扩展 3 页后才能再次执行全量；当前有效历史证据为 `p16-rules-v2`，holdout 明确属于非盲回归。
