# 分类边界与不变量

## 正类

页面同时存在主体实际图表和主体结构化表格。图表包含柱、线、点、扇区、轴、图例或数据视觉之一；表格能从 PDF 直接事实建立行、列、单元格、规则或重复数值列证据。两者都必须是页面信息表达的一部分，不能只是装饰性小图标或普通段落的偶然对齐。

## 非本类

- 只有图表，没有主体表格：交给 `body.chart`；
- 只有结构化表格，没有实际图表：交给 `body.table`；
- 图表与长正文共同占主导：交给 `body.composite.flow_text_chart`；
- 表格只存在于图片或截图内部：P16 返回能力不足，不执行 OCR；
- 非数值分类矩阵无法被当前 P6 直接证据合同识别：P16 返回表格 owner 能力不足，不猜测网格。

## 所有权

P16 在渲染前建立总分区：

1. `chart`：P13 识别的原生图表语义文字；
2. `table`：P6 单元格对象绑定后的表格语义文字；
3. `shared`：跨区域标题、单位、来源或邻近表格标题；
4. `protected`：数值、日期、代码以及不允许翻译或移动的对象。

每个源原生文字对象必须出现且只出现一次。表格对象 ID 优先纠正图表邻近启发式造成的误归属；共享 owner 必须显式建立，不能同时被图表和表格拥有。

## 不变量

1. 全页只有一个 `PageTranslationRequest`，容器 ID 全局唯一，翻译后按 owner 可逆回切。
2. 图表数据视觉、颜色映射、图例关系、轴线、点、柱、扇区和图片保持不动。
3. 表格行列、单元格归属、矢量表格线和数值锚点保持不动。
4. 图表 placement 不得侵入 table region；表格 placement 不得逃出 table region；二者不得相撞。
5. 绘图、图片、表格线和受保护对象的锁定签名在渲染前后相同。
6. P16 只执行一次合成 redaction/write，不先后生成两个叶工具候选页。
7. 任一基础 owner 能力不足时页面明确终止为 `CAPABILITY_FAILED`，不通过其他 owner 绕过。
8. 固定译文产品只能证明已给定译文的布局与渲染链；真实翻译质量必须由真实千问产品证明。
