# 工具分类与调用流程

## 组件责任

- 公共机械层：提取 PDF 事实、计算哈希、渲染预览、写 Artifact 和校验通用几何。
- P13 公开接口：识别图表区域、图表原生文字与锁定数据视觉，并提供排版和单次渲染能力。
- P6 公开接口：从直接 PDF 事实建立行、列、单元格、表格线和数值锚点。
- P16 私有层：合并两份叶模板、裁决唯一 owner、建立共享文字、构造一个页级请求、检查跨 owner 边界并给出 P16 终态。
- TranslationProvider：只返回与容器 ID 对齐的译文；不决定 bbox、字号、行距或 owner。

P16 允许调用 P6/P13 的公开模块，不读取其私有运行目录、样本特例或历史候选产物。

## 单页状态流

`source.pdf` → `PageFacts` → P13 chart template + P6 table template → P16 total ownership → 一次 `PageTranslationRequest` → provider 返回 `PageTranslationBundle` → chart/table/shared placements → 跨区域裁决 → 单次合成渲染 → 锁定视觉与质量复裁 → `PAGE_PASSED`、`CAPABILITY_FAILED`、`QUALITY_FAILED` 或 `PROCESS_FAILED`。

只有完整译文已经建立的页面才会发布候选 PDF：通过页是产品候选；已有完整译文的失败页是翻译诊断候选；模板、provider 或提前流程失败且没有完整译文时不发布 `candidate.pdf`，并在 `candidate_artifact.json` 中记录 `NO_TRANSLATED_CANDIDATE`。失败件不得计入产品通过。provider 在批次开始前缺少配置且尚未建立单页运行包时，批次只留下 `batch_result.json`，不会伪造翻译结果。

## 穿刺节奏

1. 初始 1 页：用一份人工核对的固定真译文打通所有权、页级请求、排版、渲染和视觉核查。
2. 扩展 3 页：选择中英双向、图表数量、表格形态和页面密度差距较大的页面；只接受可泛化结构修复。
3. 冻结：保存样本分组、规则说明、Prompt 和运行依赖的 SHA-256。
4. 非 holdout 回归：冻结前验证 development + regression，失败只能由一般结构证据修复。
5. 最终全量：冻结后一次访问 holdout；记录通过、能力不足和质量失败，禁止针对 holdout 改规则后仍称盲通过。

## 命令

在项目根目录先设置 `$env:PYTHONPATH='src;.'`。入口为 `python -m toolboxes.body.composite.chart_table.tools.run`。

选择参数为 `--initial`、`--initial-expansion`、`--non-holdout`、`--all` 或重复的 `--sample-id`。provider 为 `fixed`、`qwen` 或 `recorded`。任何包含 holdout 的选择都必须同时提供 `--allow-holdout --final-validation`。

## 主要 Artifact

- `input/page_facts.json`、`input/page_template.json`、`input/translation_request.json`；
- `output/translation_bundle.json`、`output/layout_plan.json`、`output/candidate.pdf`；
- `previews/source.png`、`previews/candidate.png`、`previews/comparison.png`；
- `reports/ownership_audit.json`、`reports/layout_rule_trace.json`、`reports/render_evidence.json`、`reports/candidate_artifact.json`、`reports/quality_decision.json`、`reports/run_result.json`。
