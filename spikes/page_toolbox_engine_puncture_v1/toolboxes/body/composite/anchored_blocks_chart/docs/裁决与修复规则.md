# 裁决与修复规则

- P11/P13 叶模板无法建立、任一子布局溢出、字体缺字或安全删除区不存在：`CAPABILITY_FAILED`。
- 锁定图形、保护文字或允许区外像素发生变化：`PROCESS_FAILED`。
- 译文合同不完整、占位、源语言残留，或跨 owner 译文字形碰撞：`QUALITY_FAILED`。
- 真实 provider、所有过程/能力/产品检查均通过：`PAGE_PASSED`。
- 固定机械 provider 即使全部机械检查通过，也只能是 `MECHANICAL_PASSED / NOT_EVALUATED`。

修复顺序固定为：叶工具有限排版修复 → 重新检查叶边界 → 检查跨 owner 碰撞 → 单次渲染 → 重抽事实复核。任何修复后必须重新运行块、图表、跨区和锁定对象四类检查。P15 不允许通过移动图表、扩大块边界或借用另一 owner 的空白来换取通过。
