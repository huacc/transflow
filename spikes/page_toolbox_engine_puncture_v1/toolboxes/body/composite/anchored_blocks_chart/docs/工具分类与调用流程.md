# 工具分类与调用流程

1. `shared_pdf_kernel.facts` 提取机械事实。
2. P11 模板构建器建立块 owner、块内容器和边界。
3. P13 模板构建器建立图表区域、语义容器、锚点与锁定对象。
4. P15 模板构建器先选择明确的图表语义文字和共享页眉页脚，再把未占用、可译的 P11 对象分给锚定块；其余对象保护。
5. P15 生成一个按整页阅读序排列的翻译请求。ID 形如 `anchored::<leaf-id>`、`chart::<leaf-id>`、`shared::<leaf-id>`。
6. 返回包按前缀回切给 P11/P13 布局器；两边分别只在自己的安全区内修复。
7. P15 检查跨 owner 实际字形碰撞，再把两个叶计划转换成一个渲染合同。
8. 复用 P13 的锁图渲染内核一次性删写，随后重抽事实并复核锁图、保护文字、源文残留、字形边界、锚点、字体和区外变化。
9. 过程、能力、产品三层裁决写入 `reports/quality_decision.json`；所有运行都留下候选 PDF 和视觉对照。
