# body.chart：分类边界与不变量

## 正例

- 页面主体由柱、线、点、扇区、环、坐标轴、刻度、图例和数据标签构成，图表承担主要信息表达；
- 图表可以是 PDF 矢量、整图栅格，或二者混合；
- 图表标题、语义图例、轴标题、类别标签和注释存在可定位 PDF 原生文字。

## 反例

- 连续正文占主导：交给 `body.flow_text.single` 或 `body.flow_text.multi`；
- 结构化大表与图表分别形成不可忽略区域：应进入相应 composite 叶子，而不是把整页文字都归给图表；
- 只有图片文字、没有可定位原生语义文字：本工具箱不做 OCR 或图片覆写；
- 仅有装饰性图形、没有图表结构证据：能力不足。

## 可翻译对象

- 与图表区域有稳定几何关联的原生标题、语义图例、轴标题、类别标签和注释；
- 一页所有可翻译容器组成一个 `PageTranslationRequest`；
- 大写语义词仍需翻译，不能因 `MALE`、`FEMALE` 等形式被误判为单位。

## 保护对象

- 数值、百分比、年份刻度、数据标签、页码、页眉页脚和标准单位记号；
- 图片内部文字；
- 柱、线、点、扇区、环、轴、刻度线、图例色块、颜色、边界、图片、背景和页面装饰；
- 没有目标语原生语义文字的页面必须字节级透传，Provider 不得被调用。

## 不可动约束

- 图表视觉对象签名必须保持；
- 每个原生文字对象恰有一个 owner：可翻译容器或保护集合；
- 可写槽位两两不重叠，译文字形只能落在所属槽位内；
- 图例译文不能跨过色块或改变相对关系；
- 只允许当前槽位内的字号、行距和对齐适配，不移动图表构件、不借用相邻槽位。

## 能力不足条件

- 未找到图表视觉区域或可定位原生文字；
- 目标字体缺字或未嵌入；
- 最低可读字号下仍无法装入安全槽位；
- Provider 未能返回完整真实译文或破坏必保留字面量；
- 在不覆盖图片/矢量构件的前提下无法落字。

## 当前已知边界缺口

- 中译英的中文年份字面量保护与源语言残留校验发生冲突；
- 长英文类别标签可能进入环图或图片区域；
- 图表与大表格混合页的非图表文字所有权尚未可靠排除。
