## Product PASS

- Process PASS，候选 PDF 存在；
- 图表视觉签名、保护文字、数值、年份、单位、页眉页脚不变；
- 译文字形与计划一致，位于所属安全槽位内；
- 无原文残留、占位符、图像/矢量碰撞、槽位溢出或图例错配；
- 并排预览可读且语义关系保持。

## 允许的修复

- 源字形 bbox 只提供锚点和初始排版事实，不是译文宽高的硬边界；
- 按 `源框 -> 同行安全扩宽 -> 安全区域内多行换行 -> 同类字体统一降档` 的有限规则顺序试排；
- 横向扩宽和纵向换行必须停在页面边界、相邻文字或锁定图形之前，左/右/中心语义锚点保持；
- 每个容器的规则事实、修复原子和绑定工具写入 `layout_rule_trace.json`；
- 必保留字面量可在 Provider 定点重试中临时替换为机器标记，返回后逐字符还原；
- 定点重试只请求失败 ID，每次返回后重新执行同一翻译门禁，最多重试两次；
- 重叠同文 PDF 对象可合并为一个语义 owner，但删除时必须覆盖全部别名对象；
- 同一视觉行内因中英文脚本或字体变体拆开的片段，仅在基线、颜色、字形风格、字号比例和间距同时兼容时合并；
- 图表中部的短侧标签可向外侧扩宽；靠近图表上、下边缘的叠加标签保持源锚点，避免误伸入相邻图形；
- 图表轴/类别标签和图例按语义组统一降档，允许有限的 65% 源字号比例，但仍受绝对可读字号和碰撞门禁约束；
- 相邻安全槽位可按原始对象中点裁切，裁切后仍无法互斥则能力失败。

布局门禁失败但已有合格译文时，`candidate.pdf` 必须删除对应源文并写入全部译文；允许它作为明确失败的诊断候选暴露碰撞或过密问题，但不得遗漏失败容器、保留源文或扩展页面尺寸。

## 禁止的修复

- 不移动、缩放或重绘柱、线、点、扇区、环、轴、色块和图片；
- 不 OCR、不覆盖图片文字；
- 不按样本 ID、文件名、标题、年份或绝对坐标写分支；
- 不把 placeholder、缺字或原文残留当作 PASS。
- 不裁决术语或措辞精度；但长文本仅返回极短目标语片段时，不得当作有效翻译。
