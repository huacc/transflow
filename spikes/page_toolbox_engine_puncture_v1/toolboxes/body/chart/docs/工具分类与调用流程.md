# body.chart：工具分类与调用流程

## 工具清单

| 工具 | 输入 | 输出 | 失败语义 |
|---|---|---|---|
| `extract_page_facts` | 单页 PDF | 原生文字、图片、绘图、字体和锁定哈希 | 事实提取异常为过程失败 |
| `build_chart_template` | `PageFacts` | 图表区域、语义容器、保护对象和互斥槽位 | 无直接证据或 owner 不完整为能力不足 |
| `build_chart_translation_request` | 模板、语言方向 | 一页一个翻译请求或透传 | 图片文字和保护对象不得入请求 |
| `ValidatedQwenRetryProvider` | 页级请求 | ID 对齐译文 | 占位符、残留或必保留字面量失败为 Provider 能力不足 |
| `plan_chart_layout` | 模板、译文、字体 | 锚点保持的扩宽/换行/统一字号布局计划 | 页面安全区耗尽仍溢出为能力不足 |
| `layout_rule_trace` | 模板、布局计划 | 逐容器规则研判与工具分发证据 | 不改变布局，只暴露所选规则和修复原子 |
| `render_chart_candidate` | 原页、模板、计划 | 候选 PDF、预览和 Finding | 锁定对象或保护文字变化为过程失败 |
| `_decision` | 全部 Finding | Process/Product 双结论 | 碰图、碰线、错关联等为产品失败 |

## 固定流程

```text
只读样本快照与哈希
-> PageFacts
-> 栅格/矢量图表区域
-> 原生文字逐一归属
-> 语义容器与两两互斥安全槽位
-> 页级 TranslationRequest（图片文字、数值和单位不进入）
-> Provider + 定点重试 + 必保留标记还原
-> 字面量、占位符和源语言残留校验
-> 源框、同行安全宽度和安全多行区域逐级试排
-> 同类正文、表格正文、表头分别使用统一的源字号比例档位
-> 定点删除原文字并回填译文
-> 布局失败但译文有效时仍生成不扩页的全译文诊断候选
-> 视觉签名、保护文字、实际字形、图像/矢量碰撞和图例关联复裁
-> Process/Product 双结论
```

## 本轮验证顺序

1. `CH_EN_03337_p008` 单页固定译文与千问；
2. 三页扩展：中译英、长注释、多面板柱图与环图；
3. 运行时和提示词哈希冻结；
4. 用冻结实现一次性运行 30 页，含 6 页 holdout；
5. holdout 结果只形成 Finding，不反向修改本轮实现。
