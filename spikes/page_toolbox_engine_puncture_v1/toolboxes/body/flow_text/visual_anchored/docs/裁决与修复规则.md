# body.flow_text.visual_anchored：裁决与修复规则

## 分类专用 Finding

| Finding | 严重度 | 直接证据 | owner |
|---|---|---|---|
| `VISUAL_SLOT_AMBIGUOUS` | HARD | 原生文字无法唯一绑定视觉背景 | template builder |
| `VISUAL_SLOT_OVERFLOW` | HARD | 实际译文字形或写入框越出安全槽 | layout/judge |
| `VISUAL_BACKGROUND_MISMATCH` | HARD | 槽背景采样或归属变化 | template/judge |
| `VISUAL_CONTRAST_LOW` | HARD | 保持源颜色与固定背景时，源槽对比度低于冻结的 `1.5` 可见性下限 | layout/judge |
| `VISUAL_LOCKED_OBJECT_CHANGED` | HARD | 图片、面板、颜色、边界或层叠证据变化 | renderer/judge |
| `VISUAL_SUBJECT_OCCLUDED` | HARD | 候选新增与非槽图片主体重叠 | judge |
| `VISUAL_READING_ORDER_CHANGED` | HARD | 容器 ID 或几何阅读顺序变化 | judge |
| `VISUAL_SOURCE_RESIDUE` | HARD | 候选实际字形仍含被替换原文 | renderer/judge |
| `VISUAL_TRANSLATION_MISSING` | HARD | 候选槽内实际字形与计划译文不一致 | renderer/judge |
| `VISUAL_GLYPH_OUTSIDE_SLOT` | HARD | 候选实际字形越出允许区域 | renderer/judge |

翻译进入布局前还必须通过三项机械校验：每个 `required_literals` 都是译文连续子串；英文目标不得残留中日韩统一表意文字；句子、括号、引语以及末尾冠词/介词不得结构性截断。大写 `A` 只有在其本身是必保留区域标识时才允许作为句末。

## Product PASS

流程完整不是产品通过。只有译文真实且完整、全部槽位硬门禁通过、固定视觉对象保持，并且源/候选 PNG 人工复核确认可读、无残留、无越槽、无遮挡时，产品才可 `PASS`。

## CAPABILITY FAILED

无唯一槽位、无安全擦除方式、字体缺字、对比度无法保持，或在 `max(5.5pt, 源字号×0.68)` 下仍无法槽内 fit 时，终态为 `CAPABILITY_FAILED`（产品状态等价于计划中的 `S_FAIL_CAPABILITY`），不得把文字移到页面空白处。

## Repair 允许范围

只允许依序尝试：保持源字号与节奏 → 收紧行距 → 在已证明安全的同面板/同通道宽度内减少不必要换行 → 槽内换行 → 有下限的字号缩减。允许区域可以按对齐锚点扩展到安全通道，但禁止越过相邻文字、图片、下一段、页边距或视觉主体；背景、颜色、z-order、图片和页面几何不可改变。

## Repair 接受

目标 Finding 明确改善、没有新增硬 Finding、固定对象保持、实际字形通过槽位和对比度检查、完整重渲染与复裁通过时才接受；否则回滚。连续无改善或达到有限 profile 后停止。

自动 `PAGE_PASSED` 仍不替代人工产品复核。人工必须检查目标语重复、残留普通外文词、姓名/职务重复、页眉页脚翻译、视觉锚点和阅读顺序；冻结 holdout 中任何人工失败都会使 P12 Gate 失败。
