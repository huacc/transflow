# body.flow_text.visual_anchored：分类边界与不变量

## 正例

- 大面积固定图片、主视觉或有色面板占页面主体；
- 页面只有一组或少数组连续正文，且正文在视觉上属于明确面板或留白槽；
- 正文是可定位、可独立删除和重写的 PDF 原生文字。

## 反例

- 纯照片且没有实质原生正文：`visual_only`；
- 多个互不连续 KPI 卡片：`body.anchored_blocks`；
- 普通单栏长正文：`body.flow_text.single`；
- 需要推动图片、跨视觉区域扩容或全页正文重排才能落版的页面。

## 可翻译对象

每个逻辑连续的原生文字单元恰好绑定一个 `VisualTextSlot`。请求只包含该单元的容器 ID、原文、阅读顺序和必须保持的字面量。

自然语言页眉、页脚、标题、签名和职务属于可翻译对象。相邻中文与 `A`、`LED`、`UHPC` 等行内标识在物理证据允许时合成一个语义单元，标识作为必保留字面量；重复套印字形只保留一个语义容器，但其全部源对象仍纳入擦除责任。

## 不可翻译对象

- 图片像素内部的文字、数字和标签；
- 纯 1–3 位边缘页码、URL 和无法形成自然语言语义的纯数字或装饰性标识；
- 无法证明属于独立原生文字层的可见内容。

日期、金额、百分比和代码若位于自然语言单元内，不单独翻译其值，而是以 `required_literals` 逐字符保留在译文中。

## 不可动约束

- 页面几何、固定图片、矢量面板、背景颜色、边界和层叠顺序保持；
- 背景关系、文字颜色、对齐语义和安全内边距保持；左对齐保持左锚，右对齐保持右锚，居中内容保持面板中心；
- 译文不得越槽、跨槽串写、推动全页内容或覆盖视觉主体；
- 候选必须保持足够对比度，并以实际渲染字形复核可见性与边界。

原始文字 bbox 不是绝对宽度限制。若同一面板或同一水平安全通道有空余，译文可在不碰相邻文字、图片或视觉主体的前提下横向扩展并减少不必要换行；垂直扩展最多取原高度与约 `2.4×` 源字号中的较大值，且必须在下一对象和页边距之前停止。

## 能力不足条件

- 没有可翻译的独立原生文字；
- 无法从直接 PDF 证据确定唯一安全槽位、背景关系或前后景层级；
- 目标字体缺字，或译文在原槽内降到 `max(5.5pt, 源字号×0.68)` 后仍无法容纳；
- 保持源文字颜色和固定背景时，对比度低于冻结的 `1.5` 可见性下限；
- 无法在不改变固定视觉层的前提下安全擦除原文并写回译文。
