# body.flow_text.visual_anchored：工具分类与调用流程

## 工具清单

| 工具 | 类别 | 输入 | 输出 | 失败语义 |
|---|---|---|---|---|
| `extract_page_facts` | 公共机械采集 | 单页 PDF | 原生文字、图片、矢量和锁定对象哈希 | PDF 不可读或页身份错误 |
| `build_visual_anchored_template` | P12 私有模板 | `PageFacts`、源 PDF | `VisualTextSlot`、容器和保护对象 | 无唯一安全视觉槽 |
| 页级翻译 Provider | 公共翻译 | 容器化请求 | ID 对齐译文包 | Provider/合同失败 |
| `ValidatedQwenRetryProvider` | P12 Provider 包装 | 页级请求、Prompt | 已机械复核的真实译文 | 两轮后仍缺字面量、残留源语言或结构截断 |
| `plan_visual_anchored_layout` | P12 私有布局 | 模板、译文 | 槽内字号/行距写入计划 | 最低可读字号仍溢出 |
| `render_visual_anchored_candidate` | P12 私有渲染 | 源页、模板、计划 | 候选 PDF 与源/候选 PNG | 固定视觉变化或原文残留 |
| `judge_visual_anchored` | P12 私有裁决 | 全部结构和渲染证据 | Finding 与终态 | 硬 Finding 或证据缺失 |

## 固定流程

```text
PageFacts
→ VisualTextSlotTemplateBuilder
→ PageTranslationRequest
→ TranslationProvider
→ Required-literal / residue / completeness validation
→ SlotBoundLayoutPlanner
→ CandidateRenderer
→ Source/Candidate Render Evidence
→ VisualAnchoredJudge
→ 完整复裁
```

无运行时自由工具选择。首次 Qwen 译文未通过机械校验时，第二轮只重送失败单元，并把修复结果按原阅读顺序合并；最多两个逻辑翻译轮次，不允许工程侧猜译或直接改写语义。API Key 只通过当前进程环境变量注入，不写入运行包或文档。

布局依次尝试源字号节奏、收紧行距、换行和有下限的字号缩减。若同一面板或水平通道有已证明的空余，可按左/右/居中锚点扩展安全宽度；任何扩展都必须避开相邻文字、图片、页边距和下一段内容，并在候选上重新采集实际字形复裁。

## development 探索顺序

1. `S2P0625` 单页 tracer；
2. `EN_00468_p0010` 与 `ZH_01929_p0014` 风险扩展；
3. 主流程冻结后才运行 regression；
4. 冻结代码、Prompt 和配置后，holdout 只以 `final_validation` 目的解封。

## 流程冻结条件

三张 development 页的模板、固定译文、候选 PDF、自动 Finding 和人工 PNG 复核形成可追溯闭环；所有规则均为结构驱动且没有样本 ID、品牌名、原文词句或绝对坐标分支。

冻结后以 recorded Provider 重放 20 页 regression；验收比较译文列表、`layout_plan.json` 与候选 PNG 像素。PDF 容器二进制哈希可因保存文档 ID 不同而变化，不能代替布局与像素确定性证据。
