# body.flow_text.multi 页内排版范式与回填策略

## 1. 决策顺序

1. 工程规则按源页容器归属和纵向区间生成 `FlowBand`。
2. 千问只复核页级区段顺序和多栏区子范式；不同时判断字号、坐标或修复工具。
3. 规则与千问一致才进入回填；不一致转细粒度裁决，不猜测放行。
4. 每个区段只调用本工具箱拥有的回填策略，不运行 `single` 工具箱引擎。

## 2. 页级范式

- `multi_only`：整段为多栏。
- `single_multi`：通栏后进入多栏。
- `multi_single`：多栏后回到通栏。
- `single_multi_single`：通栏、多栏、通栏依次出现。

## 3. 多栏区子范式

### 3.1 independent_columns

各栏为独立阅读流。每栏独立纵向回填，栏宽、左锚点和栏间距不可改变。

### 3.2 paired_row_columns

左右栏按行成对表达。识别证据为当前页重复出现的跨栏行首对齐、短语义单元和悬挂续行。

回填约束：

- 同一对左右单元共享 y 起点；
- 行高取两侧译文所需高度的较大者；
- 译文较短时优先贴近源行 y，放不下时才把后续整对行向下推；
- 每个普通单元的 x0、x1 保持源 bbox，不得跨栏；
- 独立栏的相对段距修复不得进入本范式，但实际字形重叠仍为硬失败。

## 4. Run 11 验证结论

`S2P0395` 判定为 `single_multi_single + paired_row_columns`。多栏清单从粗容器重建为左右各 7 个语义单元，并采用同步成对行回填。最终过程与产品门禁均通过。
