# PDF 分类专用工具箱穿刺建设计划 v1.2

## 0. 一页结论

### 0.1 本穿刺解决什么问题

页面分类穿刺已经能够把单页 PDF 路由到相对稳定的分类叶子。下一阶段不再继续扩大全局万能框架，而是按分类叶子逐个建设、测试和冻结专用工具箱。

目标链路为：

```text
分类结果（只读）
  → 公共合同与机械内核
  → Codex 作为实验引擎探索某个分类的最佳工具流程
  → 千问按页翻译
  → 分类专用工程化回填、排版、裁决和修复
  → 固定译文回归 + 千问端到端回归 + 留出验证
  → 工具箱晋级为稳定 PageToolboxModule
  → 工具箱级全量回归和资产封版
```

### 0.2 核心实施顺序

```text
P0 计划与边界冻结
  → P1 穿刺运行基建
  → P2 公共 PDF 机械内核
  → P3 body.flow_text.single 首条纵向闭环
  → P4 body.flow_text.single 稳定与晋级
  → P5 body.flow_text.multi
  → P6 body.table
  → P7 body.composite.flow_text_table
  → P8 contents
  → P9 cover
  → P10 end
  → P11 body.anchored_blocks
  → P12 body.flow_text.visual_anchored
  → P13 body.chart
  → P14 body.diagram
  → P15 body.composite.anchored_blocks_chart
  → P16 body.composite.chart_table
  → P17 body.composite.flow_text_chart
  → P18 body.composite.flow_text_diagram
  → P19 body.freeform
  → P20 全工具箱回归与资产封版
```

任何阶段没有通过自己的 Gate，不得把“已有候选 PDF”“脚本可运行”或“过程完整”当成阶段完成。

### 0.3 当前工程验收状态（2026-07-13）

本计划从本次验收起区分“工程验收”和“形式晋级”：工程验收表示当前实现、已生成产品和已声明能力边界获得用户接受，可以交接或继续后续穿刺；形式晋级仍要求补齐原 Gate 中尚未执行的证据、`stage_gate.json`、workflow freeze 和 `PromotionManifest`。不得把未执行项补写为已执行。

| 阶段 | 工具箱 | 工程验收 | 已知边界 | 形式晋级 |
|---|---|---|---|---|
| P4 | `body.flow_text.single` | **通过，用户验收** | 全量 278 页中 255 页机械产品 PASS；22 页能力/流程失败和 1 页产品失败保留为显式例外 | 待补 |
| P5 | `body.flow_text.multi` | **未通过，继续收口** | 最新批次产品通过面仍不足 | 未开始 |
| P6 | `body.table` | **通过，用户验收** | 当前能力聚焦具有稳定重复数值列直接证据的财务表格；修复后未按本次明确范围重跑全量 20 页 | 待补 |

P4、P6 的工程验收不追溯伪造缺失证据，也不自动解除 composite 对 P5 的依赖。后续若要进入正式引擎集成，先在本穿刺内单独完成形式晋级。

### 0.4 本轮最终成功标准

- 分类穿刺保持只读、独立和可复现；
- 公共层只保留跨分类稳定的机械原语和窄合同；
- 每个支持的分类都有独立工具箱、流程文档、测试样本和质量证据；
- 每个工具箱能明确返回通过、能力不足或质量失败；
- 千问按页翻译，翻译模块保持简单、独立、可替换；
- 分类树中的每个可处理叶子都有单独阶段、单独代码目录和单独 Gate；
- `visual_only` 由 P2 透传合同处理，`body.freeform` 由专用终态工具箱诚实返回能力不足；
- 全部工具箱形成可供后续独立集成计划消费的合同、版本和证据；
- 不支持的分类显式失败，不由万能工具或自由 Prompt 猜测处理。

### 0.5 明确不做

- 不修改 `page_classification_engine_puncture_v1`；
- 不重建页面分类树；
- 不在本穿刺中集成分类引擎与工具箱引擎；
- 不建设运行时 Catalog、文档级协调器、跨页样式记忆或多页拆分合并；
- 不建设通用排版规划器；
- 不建设开放式全局 ToolRegistry；
- 不建设术语库、翻译记忆、多模型路由或复杂翻译平台；
- 不翻译、不重排图片内文字；
- 不为文件名、样本 ID、页码或某个绝对坐标写特例；
- 不一次性生成全部空工具箱目录；
- 不在证据不足时宣称某个工具箱稳定。

## 1. 实施前锁定决策

### 1.1 上游与工作目录

本穿刺根目录：

```text
spikes/page_toolbox_engine_puncture_v1/
```

只读上游：

```text
spikes/page_classification_engine_puncture_v1/
```

主要输入：

```text
spikes/page_classification_engine_puncture_v1/分类结果/
```

本穿刺不得修改上游分类代码、Prompt、定义、Artifact 和分类结果。选中的页面必须复制为本穿刺的版本化样本快照，并记录来源与哈希。

### 1.2 页面分类树

正式工具箱路径：

```text
cover
contents
end
body.flow_text.single
body.flow_text.multi
body.flow_text.visual_anchored
body.table
body.chart
body.diagram
body.anchored_blocks
body.composite.flow_text_table
body.composite.anchored_blocks_chart
body.composite.chart_table
body.composite.flow_text_chart
body.composite.flow_text_diagram
body.freeform
```

特殊路径：

- `visual_only`：由 P2 公共机械内核执行原页字节级透传和不可变校验，不另建翻译排版工具箱。
- `body.freeform`：分类引擎无法稳定落入普通叶子时的确定性终态；本穿刺为其建设独立能力不足处理工具箱，不建设万能排版 fallback。

### 1.3 翻译边界

翻译以单页为单位：

```text
PageTemplate
  → PageTranslationRequest（本页全部可翻译原生文字，带容器 ID）
  → 千问一次处理本页
  → PageTranslationBundle（ID 对齐的页级译文）
  → 工程代码按容器 ID 回填
```

约束：

- 不合并不同页面为一次翻译请求；
- 不建设跨页翻译记忆；
- 千问只返回译文，不决定 bbox、字体、字号、行距和排版；
- 首版只检查返回数量、ID 对齐、空译文和必要数字保护项；
- 后续同事的翻译实现只要遵守同一接口即可替换当前千问适配器。

### 1.4 Codex 的阶段性角色

实验阶段由 Codex 作为实验引擎：观察失败、调用当前分类允许的工具、补工具和调整固定流程。工具箱晋级后，主流程必须固化为确定性状态机；正式运行不再由 Codex 在全局工具集中自由选择下一步。

### 1.5 Gate 纪律

每个阶段必须保存：

- 阶段输入；
- 实际交付物；
- 自动测试结果；
- 人工视觉核查清单；
- 未解决问题；
- `stage_gate.json`；
- 进入下一阶段的明确结论。

Gate 只有三种结论：

```text
通过              可以进入下一阶段
证据不足          当前能力保留，不得称稳定；可转去补样本或处理其他叶子
失败              必须修复本阶段问题，不得进入依赖它的阶段
```

## 2. 目标系统组成

### 2.1 公共层与私有层

```text
ToolboxPunctureRuntime              只运行当前明确指定的工具箱
├─ TranslationProvider              独立的页级千问适配器
├─ SharedPdfKernel                  机械 PDF 原语
├─ StateAndArtifactRuntime          状态、证据和运行产物
└─ PageToolboxModule                每个分类独立
├─ PageTemplateBuilder
├─ TranslationRequestBuilder
├─ LayoutPlanner
├─ ToolboxJudge
└─ RepairPolicy
```

本穿刺不根据分类结果自动选择工具箱。样本快照在各叶阶段显式指定，运行时路由和文档级装配属于后续独立集成计划。

### 2.2 可以共享的内容

- 工作区边界和源文件只读检查；
- PDF 页框、对象、文字、字体和图片事实提取；
- 源页、候选页和区域截图渲染；
- 文件和对象哈希；
- 字形覆盖与字体嵌入检查；
- PagePatch 的机械应用；
- 通用溢出、越界、遮挡和不可动对象校验；
- Artifact、状态轨迹和运行报告写入；
- 修复接受、回滚和预算控制。

### 2.3 必须由分类工具箱私有的内容

- 页面画像、正例和反例；
- 文字容器识别和翻译单元绑定规则；
- 排版参数选择和调整顺序；
- 分类特有 Finding；
- 分类特有修复策略；
- 分类特有晋级样本和质量标准。

允许不同工具箱出现重复实现。只有经过至少两个工具箱验证、且不包含页面语义的机械能力，才允许下沉到公共层。

### 2.4 核心数据合同

P1 必须冻结以下最小合同，后续阶段只能做向后兼容扩展：

| 合同 | 责任 |
|---|---|
| `SampleManifest` | 样本来源、哈希、原文档、页码、用途和标签 |
| `PageFacts` | 可复核的 PDF 直接事实 |
| `PageTemplate` | 当前工具箱对可编辑文字容器的理解 |
| `PageTranslationRequest` | 一页一次的翻译输入，包含容器 ID 和有序原文 |
| `PageTranslationBundle` | 与请求 ID 对齐的页级译文 |
| `PagePatch` | 允许修改的内容和排版动作 |
| `Finding` | 问题类型、位置、严重度、证据和 owner |
| `PageQualityDecision` | 通过、可修复、能力不足或质量失败 |
| `RunManifest` | 工具、Prompt、模型、版本、状态和 Artifact |
| `PromotionManifest` | 工具箱晋级版本及完整证据引用 |

### 2.5 中文执行状态

| 中文状态 | 内部值 | 下一步 |
|---|---|---|
| 样本已准备 | `SAMPLE_READY` | 提取事实 |
| 页面事实已就绪 | `FACTS_READY` | 构建模板 |
| 页面模板已就绪 | `TEMPLATE_READY` | 构建页级翻译请求 |
| 译文已就绪 | `TRANSLATION_READY` | 生成排版补丁 |
| 排版计划已就绪 | `PATCH_READY` | 渲染候选页 |
| 候选页已就绪 | `CANDIDATE_READY` | 执行裁决 |
| 质量裁决已完成 | `QUALITY_DECIDED` | 通过、修复或失败 |
| 局部修复中 | `REPAIRING` | 重新渲染和完整复裁 |
| 页面通过 | `PAGE_PASSED` | 写入当前工具箱回归证据 |
| 能力不足 | `CAPABILITY_FAILED` | 停止并记录缺失能力 |
| 产品质量失败 | `QUALITY_FAILED` | 停止或回到工具箱研发 |
| 流程契约失败 | `PROCESS_FAILED` | 停止并记录状态、合同或证据失真 |

## 6. P1——穿刺运行基建

### 6.1 阶段目标

建立所有工具箱都会依赖的最小运行骨架，但不实现任何分类专用排版算法。

### 6.2 任务

1. 建立项目入口、配置和测试目录；
2. 建立样本快照器和 `SampleManifest`；
3. 冻结 §2.4 最小合同及 Schema 校验；
4. 建立中文状态机和合法状态转换；
5. 建立 Run 目录、Artifact 索引和失败恢复记录；
6. 建立最小 `TranslationProvider`：一页一次调用千问；
7. 建立固定译文 Provider，用于排版确定性回归；
8. 建立 API Key 环境变量读取，禁止写入 Artifact；
9. 建立版本记录：分类树、合同、工具、Prompt、模型和参数；
10. 建立过程结论与产品结论两个独立输出。

### 6.3 最小纵向切片

```text
复制一个单页样本
  → 写入 SampleManifest
```

## 28. 风险与处理

| 风险 | 影响 | 处理 |
|---|---|---|
| 过早抽公共能力 | 改一处影响多个工具箱 | 默认私有，跨两个 leaf 验证后才下沉 |
| 千问译文波动 | 无法判断排版回退 | 固定译文与真实千问分轨 |
| rare leaf 样本不足 | 虚假稳定 | 标记证据不足，不晋级 |
| 单页样本缺少文档上下文 | 无法在本穿刺验证跨页样式一致性 | 保存文档血缘并明确列为集成计划责任，不在叶工具箱中猜测跨页状态 |
| Codex 长期充当自由引擎 | 运行不可复现 | 晋级前固化确定性流程 |
| composite 变成万能编排器 | 重走旧框架问题 | 仅允许有限、显式、leaf 私有组合 |
| 修复循环无边界 | 越修越坏或耗尽预算 | 最大轮次、无改善终止、完整复裁与回滚 |
| 字体缺字 | 乱码或视觉假通过 | 字形覆盖与字体嵌入硬门禁 |
| 密钥进入仓库 | 安全风险 | 环境变量，本地忽略，报告脱敏 |
| 候选存在即宣称成功 | 产品质量失真 | 过程与产品双裁决 |
