# P15 `body.composite.anchored_blocks_chart` 新增经验

## 1. 两个叶工具不能顺序改写同一 PDF

P11 与 P13 的模板都引用原始 `TextObjectFact.object_id`。若先用 P11 保存候选，再让 P13 在候选上查原对象 ID，第二步面对的已是重写后的内容流，对象身份不再可靠。稳定做法是：两个叶工具只产模板和布局计划，P15 把计划转换为一个渲染合同，对源页只做一次 redaction/write。

## 2. 先冻结对象所有权，再发统一翻译请求

每个原生文字对象必须恰好属于 `anchored`、`chart`、`shared` 或 `protected`。统一请求 ID 带 owner 前缀，返回后再恢复叶容器 ID。这样可以同时证明全页无重复请求、结果可逆切片，也避免两个叶工具各自翻译同一标题。

## 3. partial 容器必须按对方的 `allowed_bbox` 裁剪

P13 认领 P11 容器中的部分对象后，不能让剩余 P11 容器继承原整块安全框。只参考对方源字框也不够，因为 P13 合法译文区可能比源字框更大。P15 最终按 P13 `allowed_bbox` 裁剪 partial P11 安全框，保留 2pt 跨 owner 间距，并允许译文只在原 P11 block owner 内部错开。

## 4. KPI 货币单位不是天然的图表轴标签

`RMB`、`HK$ million` 等短单位在几何上靠近图形时容易被 P13 识别为轴标签，但若它与同一 P11 block 中的 KPI 数值紧邻，应留给锚定块。判断必须同时要求“纯单位文本 + 同 block 有近邻数值”，不能按某份样本或某个坐标硬编码。

## 5. 没有独立 P11 owner 时能力应失败，但其他 owner 仍要翻译

全量中部分页面的全部可译文字都由图表及图例语义占有，P15 返回 `P11_TRANSLATABLE_OWNER_NOT_FOUND`。这类页面更接近纯 `body.chart`，P15 不用自由 Prompt 造一个锚定块，也不从图表 owner 抢文字；但缺少 P11 owner 只是分类能力 Finding，不能阻断已经明确归属 chart/shared 的翻译与候选渲染。

## 6. 子工具失败必须原样传播

全量机械回归中仍有 P11 微型注释溢出、P13 图表槽位溢出、图例方向变化、图形碰撞和锁图签名变化。P15 只修复自己的分区和跨区问题，不通过降低字号下限、移动图表或放宽 P13 锁图规则掩盖叶工具边界。

## 7. 固定译文只证明机械链路，真实模型仍要单独过语义合同

固定 `译`/`T` 只验证所有权、ID 回切、布局、字体、单次渲染、锁图和裁决；即使机械通过，产品裁决也必须是 `NOT_EVALUATED`。真实模型接入后，还要在排版前独立检查必保字面量、源语言残留、占位符和明显不充分输出，并只定点重试失败 ID。模型调用成功、翻译包可解析或候选 PDF 已生成，都不能代替语义与产品门禁。

## 8. 每个失败页仍应留下真实译后候选

源页复制件不能叫候选。模板能力 Finding 不得阻断其他明确 owner；正常槽位无法适配的译文也不能被过滤，而应在原单页内诊断性缩排物化，并继续保留溢出、碰撞等失败终态。模型超时或临时 503 可以有限重试；重试耗尽的运行应标记 `NOT_PRODUCED` 并由批次补跑，不能用源页复制件凑齐数量。最终交付只有在每页都存在真实译后 PDF 时才完整。

## 9. 拉丁短标识不能一律当漏译，也不能一律保护

图表比较运算符和紧随数值的量级后缀属于结构字面量，可以在统一请求中显式保护；这不等于把全大写标题全部排除翻译。技术缩写、产品代号或专名若在主请求和两次定点重试中都原样保留，且只由少量首字母大写或全大写词组成，可以记录为三次确认的保留名。未达到三次一致、普通全大写标题或正文中的源语言残留仍是硬失败。

一句话原则：

> 用结构事实和重复模型证据确认保留名，不能靠“看起来像英文缩写”直接放行。

## 10. 失败候选索引应同时证明“确实翻译”和“为何失败”

仅统计 PDF 数量或失败码都不足以支持人工纠偏。最终验收包应按终态列出每页 `candidate.pdf`、源候选对照图、Finding 和 owner，并同时核验：候选可打开且保持单页原尺寸、候选不与源文件字节相同、翻译包中的每个译文都能从 PDF 提取、unfit 译文没有被省略。PDF 字体可把 `U+2022 BULLET` 提取为等价的 `U+2219 BULLET OPERATOR`，可在审计中显式归一；数值量级或单位变化不是字形等价，必须继续作为硬失败传播。PDF 存在与产品通过必须继续分开。

## 11. 当前结论

最终真实模型聚合证据为：8 `PAGE_PASSED`、14 `CAPABILITY_FAILED`、6 `QUALITY_FAILED`、2 `PROCESS_FAILED`；30/30 有真实译后候选且通过语义校验，1095/1095 个译文单元可提取，228 个 unfit 译文已诊断性物化，省略数为 0。当前已知分类池仍存在明确的叶能力、跨 owner、图形碰撞和锁图签名失败，因此译后 PDF 交付合同为 `PASS`，P15 产品 Gate 仍为 `FAIL`，不生成 `promotion_manifest.json`。前置 Gate 未正式晋级和样本非盲是额外阻塞项，不改变当前产品门禁本身失败的事实。
