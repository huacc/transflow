# P12 `body.flow_text.visual_anchored` 新增经验

## 1. 收录边界

本文只记录既有经验文档尚未覆盖、且在视觉锚定正文流的多轮验证中暴露出的两类共性问题。不记录实现方式、运行编号、样本文字、公司名、页码或绝对坐标。

## 2. 双语对照的去重不能只看几何距离

### 场景

页面同时保留源语言正文和目标语言对照正文。两者可能上下相邻，也可能分列、分区，甚至隔着图片或装饰元素。姓名与职务还可能被拆成不同文字对象，但共同构成一个语义身份。

### 容易出现的误判

- 只把紧邻、同行或样式完全一致的两段文字视为双语对照；
- 因为目标语言文字离得较远，就把它当成另一段独立内容；
- 分别处理姓名、职务和正文，导致其中一部分被重复翻译；
- 只检查原文是否被替换、文字是否越界，没有检查候选页是否出现同义内容两份并存。

### 新认识

双语对照关系首先是语义关系，其次才是几何关系。判断是否已有目标语言对照时，应综合比较内容职责、阅读顺序、标题或正文层级、姓名与职务的归属，以及它们在同一视觉叙事中的对应关系。距离和样式只能帮助缩小候选范围，不能单独决定是否重复。

若页面已经存在完整的目标语言对照，应保留一份可读内容，避免再生成同义副本；若只存在部分对照，则应先确认缺失的语义范围，不能整段重复补写。

一句话原则：

> 双语去重按“是否表达同一个语义单元”判断，不能按“两个文字框是否靠得近”判断。

## 3. 原版低可见性会形成忠实度与可读性的冲突

### 场景

源页文字颜色与背景非常接近，原文本身已经难以辨认。译文若完全沿用原颜色和背景，会忠实地复制低可见性；若擅自提高对比度，又会改变原设计。

### 容易出现的误判

- 只要颜色值与原文一致，就把结果判为视觉保持成功；
- 为了让译文可读，未经约定便改成高对比颜色；
- 因为文字框没有越界，就忽略肉眼几乎看不见的事实；
- 把源设计自身的问题误当成普通字号或排版问题反复调整。

### 新认识

“保持原设计”和“保证译文可读”在这种页面上不能同时自动满足。默认翻译排版不应悄悄替原设计做审美或无障碍改造，也不应把不可读结果包装成成功；应明确指出这是能力或产品策略边界。

只有在任务另行授权“提高可读性”时，才可以把增强对比度作为独立设计决策，并让使用者知道它不再是纯粹的版式保持。

一句话原则：

> 源页本身不可读时，忠实复制和可读输出是两种不同目标，不能静默替用户选择其中一个。
