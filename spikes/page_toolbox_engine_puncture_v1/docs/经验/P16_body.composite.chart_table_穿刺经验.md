# P16 `body.composite.chart_table` 穿刺经验

日期：2026-07-16（2026-07-17 补充）

## 结论先行

复合页的难点不在于“同时调用图表工具和表格工具”，而在于先建立一份全页唯一所有权，再进行一次翻译和一次渲染。若两个叶工具分别解释、分别写 PDF，邻近标题、表格行标签和图表标签会被重复拥有，跨区域碰撞也无法在写入前统一裁决。

本轮完成了 30 页版本化快照、1 页初始真实千问、3 页差异化扩展真实千问、v2 规则冻结、冻结后 30 页真实千问、recorded replay 和逐页视觉复核。全量结果为 16 `PAGE_PASSED`、13 `CAPABILITY_FAILED`、1 `PROCESS_FAILED`，但 30/30 都留下可打开、单页且尺寸不变的 `candidate.pdf`。P6/P13 仍未满足 composite 形式晋级前置，v2 holdout 也不是严格盲测，因此 Gate 继续为证据不足。

## 最有价值的结构发现

### 1. 表格对象 ID 比邻近关系更可靠

P13 面向图表时会把邻近文字合理地解释成图表标题或标签，但复合页中的多行表格行名可能紧贴图表。P16 应先让 P6 用行列/单元格直接证据建立对象集合，再从 P13 owner 中扣除这些对象。这个优先级解决的是职责冲突，不是样本补丁。

### 2. 无竖线表格需要重复行证据

只依赖矢量表格线会漏掉常见财务表。连续多行出现相同横向语义间隔时，可以恢复隐含子列；必须要求至少三行重复，避免把偶然对齐的图表标签误判为表格。

### 3. 表格尾部扩张必须同时要求列证据和数值证据

仅按纵向距离吸收尾部行，会把下方图表标题或图例吞进表格。要求“至少两列 + 至少两个数值对象”后，能覆盖币种金额尾行，同时阻止尾部图表行进入 table owner。

### 4. shared 不是剩余垃圾桶

表格上方的标题、单位、来源可能跨 chart/table 语境，但仍需由几何关系显式建立 shared owner。不能把未能解释的文字都塞进 shared，否则所有权审计形式上完整，语义上仍然错误。

### 5. 能力失败比伪成功更重要

`CT_ZH_06996_p100` 和 `CT_EN_02396_p009` 都缺少当前 P6 所要求的重复数值列直接证据，因此 composite owner 仍应能力失败。但两页都存在可提取的原生文字，失败后可以降到“原生文字诊断翻译”，完整回填译文 PDF；这不等于伪造 table owner，也不允许 OCR、伪造单元格或把表格当正文后判产品通过。

## 冻结与重放

初始和扩展页稳定后冻结 15 个运行时、依赖、Prompt、测试和样本清单文件。新增“所有终态都保留 PDF”合同后，再用 4 页真实千问 bundle 做 recorded replay：candidate PDF 字节哈希不同，但 candidate PNG 哈希逐页完全一致。PDF 差异来自序列化元数据，像素哈希才是本轮渲染确定性证据。

全量 30 页的结构范围覆盖 1–35 个图表区域、3–13 列、2–21 行、17–59 个可译容器和 14–114 个受保护对象；28 个可建模页面有 28 个不同页面结构哈希。这比只扫描运行时代码中是否出现样本 ID 更能说明规则没有绑定单一形态，但仍不能替代真实千问产品验收。

## 新增经验：失败 PDF 与产品裁决必须彻底分离

模板能力失败、provider 失败和流程异常可能发生在正式候选生成前。若只写 JSON，无法直接判断是源页分类边界、翻译输出、排版还是渲染问题。

更新后的稳定合同是：`output/candidate.pdf` 只在完整译文包已经物化后发布。翻译完成但能力、布局或质量门失败时，仍生成已翻译诊断 PDF，并记录 `diagnostic_candidate=true`、`product_acceptance=false`、终态和失败码；若 provider 在任何译文返回前失败，则明确记录 `NO_TRANSLATED_CANDIDATE` 并重跑，禁止复制源页冒充候选。所有已发布 PDF 继续验证可打开、页数和页面尺寸不变。

一句话原则：

> PDF 是问题证据，不是成功证明；但名为候选的 PDF 必须已经完成翻译。

## 新增经验：图例文字必须绑定“数值 + 色点”的行锚

同色、同字号、相邻的多行图例文字不能仅按文本样式合并。百分比和色点通常属于受保护对象，若语义文字没有绑定对应数据行，多条译文会挤进一个容器并整体上移。应先识别同一横向行中的小色块和百分比，再以该行中心约束译文；不同数据行即使样式相同也不能合并。

## 新增经验：正文 bbox 是锚，不是固定高度牢笼

长正文页若下方存在安全空白，只按源字形 bbox 排版会得到过小字号和过紧行距。更稳定的规则是保持左锚点与后续视觉障碍不变，把允许区向下扩展到最近障碍，再优先尝试较大字号和约 1.18 行距。混合 CJK/Latin 词和句尾独立百分比也应保留在同一段落容器中。

## 新增经验：数量级校验必须按完整词法单元判断

不能因为 `百万`/`百萬` 含有字符 `万`/`萬`，就强制译成 `ten-thousand`；它们正确对应 `million`。独立 `万`、数字紧邻的 `N万` 与 `亿` 才进入相应数量级守卫。类似地，`FDA`、`DFU`、`PMDA` 这类独立专业缩写可以在中译结果中保留，但不能因此放行包含多个英文词的未翻译短语。

## 新增经验：字面量失败不总等于语义数据变化

全量真实千问暴露了三类等价表达：`TOP 10` 变为中文数词、中文数字月份变为英文月份名、完整港币金额改写为“亿/万”。它们可能语义等价，逐字符串检查却会失败。

不能因此全面放宽数字保护。更安全的顺序是：

1. 纯币种金额轴标优先归入保护集合，不发送翻译；
2. 日期月份和 `TOP N` 只允许结构化、窄范围的等价校验；
3. 同页若还存在槽位溢出、跨 owner 侵入、字形越界或受保护对象变化，仍保持失败；
4. 视觉上看似正常的等价表达也不能越过当前冻结合同直接改写为 PASS。

## 新增经验：实际可见截断必须独立于字符串门禁

有一页只记录了日期字面量失败，但放大诊断 PDF 后还能看到表头英文单位在右端被截断。这说明“字符串译文完整”和“PDF 中实际字形完整”是两道不同门禁。后续应为实际插入字形、提取文本和计划译文的一致性补回归，不能用修复日期等价规则掩盖可见截断。

## 本轮全量结论

- 30/30 有已翻译候选 PDF，30/30 可打开、单页、尺寸不变，源页复制候选为 0；
- 20/30 正式产品通过，10 页保留已翻译诊断候选；
- 30/30 翻译校验通过，995/995 个译文单元完整返回；
- 30/30 v4 重放候选 PNG 与真实千问全量候选逐页像素一致；
- P1、P6、P13、P16 联合回归 116/116，通过后冻结文件哈希保持一致。

## 后续恢复点

1. 先以失败索引中的 PDF 确认修复优先级，优先补“实际可见截断但机械未报 overflow”的回归；
2. 分开处理严格字面量等价、密集表格容纳、跨 owner 侵入和受保护文字变化，不用一个宽松规则同时覆盖；
3. 两个 table owner 能力失败先决定扩张 P6 能力还是修正分类，不在 P16 内加入 OCR 或样本分支；
4. 任一运行时修改都建立新 freeze revision，并重新执行 1 页、3 页、全量；任何新运行不得恢复源页复制候选；
5. 另行补齐 P6、P13 的形式 Gate，且 P16 自身全量通过后，才生成 P16 的 `promotion_manifest.json`。
