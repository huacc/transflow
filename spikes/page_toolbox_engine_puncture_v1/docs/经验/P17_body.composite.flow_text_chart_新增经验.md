# P17 `body.composite.flow_text_chart` 新增经验

本文只记录 P17 相对 P4 正文流与 P13 图表经验的净新增结论。P4 已说明行距、段距与垂直余量的通用规则；P13 已说明源 bbox 不是普遍硬边界、图表文字需按角色分组拟合。这里不重复这些内容。

## 1. 先确定字号可行性，再用同一字号回收复合 FlowBand 的垂直余量

### 失败现象

- 首个可行 profile 为了消除溢出或坏断行降低了字号，同时也继承了紧凑行高；页面虽然机械通过，但正文仍挤在上半区，图表前留下大块无效空白。
- 正文相邻段的 `gap_scale` 未实际进入 body-to-body 间距计算，导致同页段落间距无法随 profile 一致放松。
- 若为填补空白直接上移图表，会改变 P17 锁定的图表视觉几何，破坏“只重排流式正文、图表视觉不动”的复合叶边界。

### 稳定规则

1. 先按既有 P4 profile 找到满足溢出、标点和断行门禁的字号档位。
2. 在该字号不变的前提下，计算正文末端到 FlowBand 下边界的剩余高度。
3. 只有当余量达到正文中位字号的 3 倍时，才按有限档位重试：优先 `line_height >= 1.55`、`gap_scale >= 1.35`，失败后回退到 `1.40 / 1.15`。
4. body-to-body 间距必须实际乘以当前 profile 的 `gap_scale`，使同页各段按同一节奏变化。
5. 始终保持正文左锚和图表视觉坐标不变；若放松后产生碰撞，则回退到先前可行 profile，而不是移动图表。

这解决的是“复合叶先压缩字号后仍有大量垂直余量”的阶段顺序问题，而不是为某个页面设置固定坐标。

## 2. 原文位于柱形内部时，目标文字必须继承柱形硬包络

### 失败现象

- 通用图表规则允许标签 bbox 向安全方向扩展；但原标签完整位于柱形内部时，这种扩展会让译文越出柱体，视觉语义从“柱内标签”变成“浮在柱旁的文字”。
- CJK 短标签按自然字符流换行时，容易出现 `4+1` 的孤尾；窄柱内英文即使允许在连字符处分行，也可能因相对字号下限过高而被错误判为能力失败。

### 稳定规则

1. 对 `AXIS_OR_CATEGORY_LABEL`，从局部视觉中选择完整包围源标签、面积最小且尺寸与源字号相容的视觉对象作为内部 owner。
2. 将目标 `allowed_bbox` 与该 owner 的内缩包络求交，并在源标签接近 owner 中心时保持居中；译文不得因为更长而向柱外扩展。
3. 纯 CJK 且长度不少于 5 个字符时，先尝试均衡换行，避免孤立末字；英文保留连字符处分行候选。
4. 对水平、居中、`OVERLAY` 且可用宽度不超过 4 个源字号的窄内部标签，才额外尝试 `50%` 字号档，并以 `4.5pt` 为绝对可读下限。
5. 到达绝对下限后仍不能完全放入 owner，必须硬失败，不能以越柱、裁切或拆碎单词换取机械通过。

该规则是结构关系拟合：依据角色、包含关系、相对尺寸和可读下限，不依赖样本名、固定字符串或固定页面坐标。

## 4. 复合叶必须单独验证 owner 仲裁的尺度不变量

P4 与 P13 各自通过回归，不等于 P17 的组合结果能够泛化。复合叶新增了正文候选选择、图表 owner 关闭、FlowBand 分段和 chart guard 合并；这些仲裁步骤可能在两个叶工具都能正常建模时，因尺度变化把同一原生文字从 `flow` 改判为 `chart`，或改变正文区域数量。

反过拟合探针应把页面、文字 bbox、字号、图片和绘图对象统一缩放，再比较归一化后的结构，而不是比较原始坐标或结构哈希。至少检查：

1. 每个原生文字对象的 `flow/chart/shared/protected` owner 是否不变；
2. 容器的源对象集合、角色和锚点关系是否不变；
3. FlowBand 的数量、模式和成员是否不变；
4. chart guard 与 protected 集合是否不变。

探针应先做小幅双向扰动，再逐级扩大幅度，以定位首次结构翻转。缩放倍率只属于测试探针，不能成为运行时分支。只要小幅等比变化就造成 owner 或 FlowBand 翻转，即使原尺寸全量页面全部通过，也应判为结构尺度敏感，不能继续宣称反过拟合通过。

## 5. 固定数值按下游影响分级，不按数值大小一概判断

固定数值不是天然过拟合，但必须区分它处在规则链的哪个位置：

- 浮点比较、实际字形碰撞、极小 gutter 等机械 epsilon，可以保留物理点容差；
- 字号、行高和段距的有限搜索档位，可以作为可解释的布局 profile；
- 决定 narrative lane、上下邻接、FlowBand 分段或 `flow/chart` owner 的阈值，属于语义结构阈值，不能直接依赖未经归一化的绝对点值。

语义结构阈值应由当前页字号、行步长、页宽页高比例、对象尺寸和相对拓扑推导，并通过尺度变形验证。判断标准不是“源码里有没有样本 ID”，而是该数值越过边界时是否会改变文字归属或页面结构。

## 6. 历史反过拟合 PASS 使用前必须校验冻结哈希

`workflow_frozen`、旧报告中的 PASS 和回归数量只是声明，不是当前工作树仍与当时一致的证明。引用历史证据前，必须重新计算 `rule_freeze` 跟踪的运行时、直接依赖、Prompt 和测试哈希：

1. 任一相关文件哈希不一致，历史反过拟合报告立即降为过期证据；
2. 在最终代码树上重新执行身份扫描、Prompt 扫描、尺度变形和回归；
3. 由同一棵代码树重新生成冻结记录与审计报告；
4. 冻结后的任何相关修改，都必须再次失效并刷新该证据链。

本轮再次证明，当前样本上的产品回归通过、历史反过拟合报告通过和现代码已经证明泛化，是三个不同结论，不能互相替代。
