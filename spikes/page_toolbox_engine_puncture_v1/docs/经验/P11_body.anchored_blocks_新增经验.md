# P11 `body.anchored_blocks` 新增经验

## 1. 收录边界

本文只记录 P4、P6 既有经验文档尚未覆盖、且在独立锚定块穿刺中经过多次尝试后形成的新认识。

既有文档已经讨论的单页递进、横向稳定与纵向适配、源字形 BBox 与安全区域的区别、页眉页脚语义拆分、实际字形验收、Prompt 静态反过拟合扫描等内容，不在这里重复。

本文不记录工程实现、运行编号、样本文字、公司名、页码或绝对坐标。

## 2. 新经验一：分隔线可以约束卡片，但不能据此推导表格语义

### 场景

独立 KPI 卡片经常使用横线、竖线或矩形边框。视觉上可能像网格，但各区域之间没有稳定的行列数据关系，也不能按表格连续读取。

### 容易出现的误判

- 看见竖线和横线就把页面当成表格；
- 反过来，因为分类已经是独立块，就忽略分隔线，让译文越过卡片边界；
- 把几何格子同时当成文字安全边界和数据单元格所有权。

### 新认识

分隔线在独立块中的职责只是提供局部空间边界。它可以证明“文字不能越过这里”，但不能单独证明：

- 上下区域属于同一数据行；
- 左右区域属于同一数据列；
- 某个标题统领整列；
- 数值与标签存在表格式行列映射。

只有出现稳定重复的行列关系和数据对应时，才应由表格分类处理。没有这些关系时，分隔线只参与卡片边界判断。

一句话原则：

> 线条可以证明空间边界，不能自动证明表格语义。

## 3. 新经验二：字号一致性必须跨独立块比较

### 场景

多个卡片使用同一套标题、说明和微型标签样式，但译文长度不同。若每个卡片独立求解，长译文会被单独缩小，短译文则保持原字号。

### 多次尝试后确认的问题

每个文字框都能装下，并不代表整页视觉正确。逐框独立压缩会造成：

- 同级卡片标题大小不一致；
- 相同说明样式在不同块中出现不同字号；
- 标题与正文原有的字号差被压平；
- 微型脚注反过来牵连普通正文一起缩小。

### 新认识

字号关系需要在两个层面同时保持：

1. 同源样式跨块一致：源页字号、字体角色和视觉职责相同的文字，在目标页仍应保持一致；
2. 不同样式保留差值：标题与正文、正文与脚注之间原有的层级差，不应因局部 fit 被消除。

微型文字应属于独立样式组，不能仅因处在同一块或同一页就与普通正文联动。

一句话原则：

> 独立块可以独立排版，但同一页面的样式层级不能各自为政。

## 4. 新经验三：居中应保持语义槽位中心轴

### 场景

源文字在卡片中居中，但源语言字形较短。目标语言变长后，如果仍从源文字左边界开始排，虽然文字没有越界，整段的视觉中心却会明显向一侧漂移。

### 容易出现的误判

- 把源文字 `x0` 当成必须保持的锚点；
- 只检查译文是否位于卡片内部，不检查视觉中心；
- 扩大可用宽度后仍按左对齐绘制；
- 多行译文的各行中心不一致，形成锯齿式偏移。

### 新认识

居中保存的是文字与语义槽位的中心关系，而不是源字形框的左坐标。目标语言重新换行后，应继续围绕同一中心轴组织。

判断时应比较：

- 源文字行中心相对卡片或槽位中心的位置；
- 候选各行中心是否仍落在同一视觉轴附近；
- 左右安全空间是否允许对称使用；
- 同行存在保护对象时，实际可用槽位的中心是否已经改变。

右对齐和左对齐也遵循同一思想：保持的是右锚点或左锚点，而不是机械复制整个源字形 BBox。

一句话原则：

> 跨语言保持对齐，保存的是视觉轴和边缘锚点，不是源字形的偶然宽度。

## 5. 新经验四：同一视觉行可能带有“幽灵行号”

### 场景

肉眼处于同一基线、语义连续的两个文字片段，在 PDF 抽取结果中可能拥有不同 line index；反过来，同一 block 或同一 line index 中也可能混入视觉上不连续的对象。

### 多次尝试后确认的问题

把 line index 当成硬边界，会把一个自然短语拆成多个翻译单元，导致：

- 译文语义断裂或重复；
- 同一句中的片段采用不同字号、对齐或安全区域；
- 后一个片段被误判为新的卡片说明。

### 新认识

视觉行关系应优先由当前页证据确认：

- 基线是否接近；
- 字形高度是否明显重叠；
- 水平间距是否符合当前字号；
- 字体、字号和颜色是否兼容；
- 中间是否存在保护对象、分隔线或块边界。

line index、span index 和 block index 都只能作为候选线索。它们与可见几何冲突时，应以可见几何和语义连续性为准。

一句话原则：

> PDF 行号描述导出过程，不保证描述肉眼看到的行。

## 6. 新经验五：译文完整性必须在排版之前单独裁决

### 场景

模型可能返回明显过短的长段译文、在句中途停止、留下普通源语言词、产生占位符，或把一个应翻译的字母数字序数误当成必保代码。

### 容易出现的错误处理

- 因为短译文很容易装入，就把它判成排版成功；
- 通过放大字号或增加留白掩盖译文缺失；
- 把所有源语言残留都判错，误伤合法专名；
- 把所有字母数字混合词都锁定，无法区分术语、代码和普通语义。

### 新认识

排版前应先回答四个独立问题：

1. 译文是否覆盖到原文结尾；
2. 句尾是否在语法上完整，而不是停在功能词或连接词；
3. 源语言残留是合法专名/标准编号，还是漏译；
4. 字母数字混合词在当前语义中是标识，还是应正常翻译的序数或普通词。

只有语义完整性通过，文字长度才有资格进入排版判断。

一句话原则：

> 容易放下的残缺译文仍然是失败，排版不能替翻译补语义。

## 7. 新经验六：反过拟合审计需要几何变换探针

### 场景

运行时没有样本 ID、文件名、公司名和固定句子，静态扫描全部通过，但规则仍可能只适合当前样本共同的页面尺寸和字体尺度。

### 新方法

对同一页面结构做不改变分类语义的变换，再比较结构结论：

- 等比例缩放页面、字体、文字和视觉对象；
- 整体平移一个块，同时保持块内相对关系；
- 替换实体名称，但保持文字角色和几何位置；
- 改变文案长度，但保持标题、正文、数字和单位角色；
- 改变块间空白，但不改变块所有权。

变换前后，块数量、对象所有权、保护类型、对齐和字号层级应尽量保持。若只因页面等比例缩小，某个大数字便从保护对象变成可翻译文字，说明判断依赖的是固定点值，而不是分类结构。

这类探针补足了静态扫描的盲区：

- 静态扫描回答“是否绑定样本身份”；
- 几何变换回答“是否绑定样本分布”。

一句话原则：

> 搜不到样本名字，只能证明没有身份绑定；结构经变换仍稳定，才能进一步证明没有尺度绑定。

## 8. 本文新增经验检查表

1. 分隔线是否只用于约束空间，没有被误当成表格行列语义；
2. 同源样式是否跨块保持一致，样式层级差是否仍存在；
3. 居中文字是否仍围绕原语义槽位的中心轴；
4. 视觉同行是否被错误的 line index 拆开；
5. 译文完整性是否在排版之前独立通过；
6. 静态反过拟合扫描之后，是否补做了尺度、平移或文案替换探针。
