# 工具箱统一实施节奏执行规范

## 1. 目的与边界

本规范把《分类专用工具箱穿刺建设计划》第 8 节转为可执行约束，供 P3 及之后的每个叶子工具箱复用。

它只规定工具箱如何开始、如何取样、如何冻结、如何验收和如何晋级，不包含任何页面分类专用排版策略。第 8 节验收期间不得创建正式工具箱实例。

## 2. 单一活动工具箱

- 同一时间只能存在一个处于建设状态的 `toolbox_key`；
- Gate 为 `FAIL` 时，该工具箱保持活动状态，不允许切换到下一工具箱；
- Gate 为 `PASS` 或 `EVIDENCE_INSUFFICIENT` 时可以结束当前阶段；
- `EVIDENCE_INSUFFICIENT` 必须同时给出缺失证据，且不得写入晋级清单。

## 3. 固定实施顺序

1. 用脚手架创建且只创建一个计划指定的叶子目录；
2. 冻结《分类边界与不变量》中的正例、反例和不可动约束；
3. 登记 development、regression、holdout 样本及来源哈希；
4. 仅用 development 探索 `PageTemplateBuilder → LayoutPlanner → Judge → RepairPolicy`；
5. 用固定页级译文完成确定性排版回归；
6. 用千问执行一次真实页级翻译和回填；
7. 主流程固定后运行 regression；
8. 冻结代码、Prompt 和配置，再解封 holdout，仅用于最终验证；
9. 自动 Finding 与人工视觉复核均通过后生成 Stage Gate；
10. 仅当完整验收包通过时生成 `promotion_manifest.json`。

## 4. 样本分区合同

`samples/manifest.jsonl` 每行必须包含：

- `sample_id`；
- `toolbox_key`；
- `split`：只能为 `development`、`regression` 或 `holdout`；
- `source_ref` 和小写 SHA-256；
- 原文档标识与原页码。

三个分区缺一不可，样本标识不得重复。流程冻结前禁止读取 holdout 内容；冻结后也只能以 `final_validation` 目的读取，不允许根据 holdout 结果继续调参。

## 5. 成熟度状态机

```text
EXPERIMENTAL ──→ REGRESSION ──→ PROMOTED
      │                │
      └────→ EVIDENCE_INSUFFICIENT
                       │
                       └────→ EXPERIMENTAL
```

`PROMOTED` 在本次穿刺中为终态。进入 `PROMOTED` 必须先通过验收包校验，不允许仅凭人工声明修改状态。

## 6. 最小验收包

每个工具箱必须具备：

- 三份设计文档：《分类边界与不变量》《工具分类与调用流程》《裁决与修复规则》；
- 三分区样本清单；
- 固定译文、千问页级、regression、holdout 和视觉复核五份 PASS 报告；
- 报告引用的源页、候选页、并排截图或其他真实证据文件；
- `stage_gate.json`，且决定为 `PASS`；
- 晋级时的 `promotion_manifest.json`。

所有 `evidence_refs` 必须指向工具箱包内部的真实文件，不允许路径逃逸，也不接受只有路径但文件不存在的报告。

## 7. 公共内核变更门禁

叶子工具箱需要修改公共 PDF 机械内核时：

1. 暂停当前工具箱建设；
2. 将内核修改作为独立变更验证；
3. 对所有已晋级工具箱运行回归；
4. 只有全部结果均为 `PASS` 才能恢复当前工具箱。

没有已晋级工具箱时允许继续，但仍需运行公共内核自身测试。

## 8. 执行命令

```powershell
# 预览，不写磁盘
python scripts\scaffold_toolbox.py --leaf-key body.flow_text.single --dry-run

# P3 获准后创建唯一叶子
python scripts\scaffold_toolbox.py --leaf-key body.flow_text.single

# 验收工具箱；完整晋级包缺失时返回非零
python scripts\validate_toolbox_package.py `
  --leaf-key body.flow_text.single `
  --package-root toolboxes\body\flow_text\single
```

## 9. 第 8 节本身的验收口径

- 脚手架可 dry-run，且不会提前创建 `toolboxes/`；
- 非 dry-run 只创建请求的叶子目录，不批量生成其他分类；
- 样本分区、holdout 隔离、成熟度、单一活动工具箱和公共内核回归门禁均有自动测试；
- 不完整或引用伪证据的验收包不能晋级；
- 第 8 节 Gate 通过后，才允许进入 P3。
