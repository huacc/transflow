# Transflow

Transflow 是从零建设的 PDF 翻译排版执行引擎。生产主链以完整 PDF 为输入，使用
PyMuPDF 原生能力处理页面，并在源 PDF 副本上回放已批准的页面修改；不使用
HTML、Chrome/Chromium 或页级 PDF 拼接生成最终文件。

当前施工状态为 P0：只建立可重算基线、生产包入口、测试/Gate 骨架和治理台账，
尚未实现领域合同、分类、工具箱或 PDF 运行逻辑。

## 目录边界

| 路径 | 当前职责 | 运行时边界 |
|---|---|---|
| `src/transflow/` | 新生产包的唯一源码根 | 不得运行时导入 `spikes/` 或 MerqFin 源码 |
| `resources/` | 版本化配置、Schema、Prompt 和清单 | 只允许显式登记的生产资源进入发布物 |
| `scripts/` | 开发、Gate 和迁移辅助脚本 | 不是面向用户的 CLI，不进入生产包 |
| `tests/` | 单元、合同、集成、迁移和工作流测试 | 测试材料不进入生产 wheel |
| `docs/设计/`、`docs/计划/` | 已批准设计与实施计划 | 是施工和验收依据，不是运行输入 |
| `docs/迁移/` | 来源基线、迁移台账和追溯矩阵 | 只读治理证据，不被生产代码 import |
| `spikes/` | 分类与工具箱穿刺来源 | 保持原样作为迁移证据，不直接改造成生产包 |
| `transflow-legacy-workflows/` | 已归档的旧工作流资产 | 不作为新引擎入口或依赖 |
| `样本/`、`测试数据/` | 本地测试材料 | 不得按文件名、样本 ID 或页码形成生产分支 |
| `tmp/` | 可重建的本地临时输出 | 不作为 Gate 或发布权威证据 |

## P0 验证入口

在安装开发依赖后，从仓库根运行：

```powershell
python scripts/run_gate.py G0
```

底层检查由 `resources/manifests/gate_catalog.json` 明确列出；统一入口只负责编排、
保留真实命令输出并返回准确退出码，不实现通用工作流引擎。
